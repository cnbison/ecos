## 0. CTA 数学基础的理论定位

ECOS 的 CTA 不是"用 LLM 直觉估计学生状态"，而是用心理测量学方法构建 **5D 信念分布** + BloomProfile。本文梳理 CTA 数学基础的 5 个核心理论，明确每个理论**借鉴到 ECOS CTA 的具体环节**（算法层 / 接口层 / 评估层）。

```
┌────────────────────────────────────────────────────────────────┐
│                    CTA 信念分布的数学层级                        │
├────────────────────────────────────────────────────────────────┤
│ L4 决策层（最具体）  │ Causal Inference（干预效果归因）        │
│ L3 选择层           │ CD-CAT（下一题/下一干预的自适应选择）    │
│ L2 状态层           │ MIRT（5D 状态多维联合估计）             │
│ L1 时间演化层        │ BKT / DKT（单知识点掌握度的时间演化）  │
│ L0 概率框架层        │ POMDP / HMM（统一概率状态空间）        │
└────────────────────────────────────────────────────────────────┘
```

5 个理论不是平行的——它们构成 **L0→L4 的数学栈**：从底层的概率框架，到状态估计、时间演化、自适应选择、决策归因。

## 1. 核心理论 1：MIRT（Multidimensional Item Response Theory）

### 1.1 核心观点

**MIRT**（Reckase, 2009）是 IRT 的多维扩展。在单维 IRT 中，学生的能力是 1D 标量 θ，MIRT 把它推广为 D 维能力向量 **θ = (θ₁, θ₂, ..., θ_D)**。

经典 MIRT 模型（补偿型 MIRT）：

```
P(答对题目 i | θ) = c_i + (1 - c_i) · σ( a_i · θ + d_i )
```

其中：
- `θ` ∈ ℝᴰ（D 维能力向量）
- `a_i` ∈ ℝᴰ（题目在 D 维上的区分度向量）
- `d_i` ∈ ℝ（题目难度参数）
- `c_i` ∈ (0, 1)（猜测参数）

**关键数学性质**：
- **维度相关性**：MIRT 通过 θ 的协方差矩阵 Σ_θ 表达维度间的相关结构
- **补偿型 vs 非补偿型**：补偿型 MIRT 允许"某维度弱可由另一维度强补偿"；非补偿型不允许
- **信息函数**：每个题目在 θ 空间提供"信息量"，可指导自适应选题

### 1.2 与 ECOS CTA 的对接

**CTA 的 5D 状态向量与 MIRT 自然对应**：

| CTA 维度 | MIRT 对应 | 含义 |
|---|---|---|
| K（Knowledge）| θ_K | 知识掌握度 |
| P（Procedure）| θ_P | 解题程序技能 |
| S（Strategy）| θ_S | 思维策略能力 |
| C（Confidence / 概念联结）| θ_C | 概念联结强度（含 misconceptions）|
| X（External / 元认知）| θ_X | 元认知 + 外部支持依赖 |

**关键挑战**：
- **维度相关性建模**：K 与 P 在数学学科中常正相关（K 强通常 P 也强），但在语文/英语学科中相关性弱。MIRT 的 Σ_θ 协方差矩阵能自然表达学科差异。
- **补偿 vs 非补偿**：MIRT 默认是补偿型（K 弱可由 P 强补偿答对难题）—— ECOS 是否接受这种"伪掌握"？**强烈建议 ECOS 用非补偿型 MIRT**——避免"K 弱 P 强也被判掌握"的诊断错误。

### 1.3 借鉴决策

| 决策项 | 选择 | 理由 |
|---|---|---|
| **基础模型** | 非补偿型 MIRT（Bi-factor MIRT）| 避免维度补偿造成的伪诊断 |
| **维度数** | 5D（K/P/S/C/X）+ 1 个总维度 G | 5 个特化维度 + 1 个一般维度（参考 Cai et al. 2011 的分层 MIRT）|
| **协方差结构** | 学习自学生数据，不预设 | 避免对学科差异做硬编码 |
| **CTA 集成方式** | MIRT 作为 CTA 信念分布的**生成模型层**，LLM 作为**感知层** | 详见 §7 整合 |

### 1.4 实施注意事项

- **冷启动**：MIRT 需要历史作答数据校准 a_i, d_i, c_i 参数。新学生/新学科的冷启动需要其他方法（如教师先验 + 少量预测试题）
- **可解释性**：MIRT 的 θ 估计有清晰的统计解释（z-score），可对外报告"学生在 K 维处于年级中位 +0.5σ"
- **计算成本**：MIRT 的 EM 算法在 D=5 + N=100 学生规模下，单次更新 < 1 秒

## 2. 核心理论 2：CD-CAT（Cognitive Diagnosis Computerized Adaptive Testing）

### 2.1 核心观点

**CD-CAT**（认知诊断计算机自适应测试，Cheng, 2009; Wang et al., 2011）把 CAT 从"能力估计"扩展到"认知诊断"——即估计学生是否掌握每个**属性（attribute/skill）**。

**核心机制**：
1. **属性结构**：每个题目标注它考察的属性集合（如"二次函数概念"+"分情况讨论"）
2. **Q 矩阵**：题目-属性的 0/1 矩阵（题目 i 是否考察属性 j）
3. **属性掌握模式（attribute mastery pattern）**：学生的状态是二值向量 α ∈ {0,1}ᴬ（A = 属性数）
4. **选题算法**：每次选**对当前 α 估计最有用**的题目（信息量最大 / 区分度最大）

**CD-CAT 的核心优势**（vs 传统 IRT-CAT）：
- 输出**可解释的诊断**：不仅是能力分，还有"掌握了哪些属性"
- 支持 **mastery/non-mastery 判定**（与 BloomProfile 配合）
- 短测验即可估计（10-20 题 vs IRT-CAT 的 40+ 题）

### 2.2 与 ECOS CTA 的对接

**CTA 的 BloomProfile 与 CD-CAT 的属性结构自然对应**：

| ECOS 概念 | CD-CAT 对应 |
|---|---|
| 5D 状态向量 θ | 属性掌握度（连续版）/ 二值属性 α（离散版）|
| BloomProfile（L1-L6）| 每个属性的"层级掌握模式" |
| 学习干预 | 选下一题（自适应）|
| 学科诊断报告 | "学生在属性 X 尚未掌握" |

**关键设计决策**：CTA 的"下一题选择"应**复用 CD-CAT 框架**——这与 v2.0 §3.6 的"完整 ECOS 数据流"完全契合。

### 2.3 借鉴决策

| 决策项 | 选择 | 理由 |
|---|---|---|
| **核心算法** | GDINA（Generalized DINA, de la Torre, 2011）| DINA 的最一般化扩展，兼容多种属性组合关系 |
| **选题策略** | PWKL（Posterior Weighted Kullback-Leibler, Cheng, 2009）| 兼顾信息量与诊断明确性 |
| **停止规则** | 属性层 SE ≤ 0.3 + 题目数 ≤ 20 | 兼顾诊断精度与用户体验 |
| **Q 矩阵** | 学科教师 + CTA 联合构建 | 教师提供先验，CTA 验证 |
| **冷启动** | 默认 Q 矩阵（每个知识点对应 1-3 个属性）| 后续用数据修正 |

### 2.4 实施注意事项

- **Q 矩阵构建是核心难点**：错误的 Q 矩阵会导致所有诊断偏移。建议 Phase 4 MVP 阶段先在初中数学 1 个单元验证 Q 矩阵构建流程
- **属性数膨胀**：避免属性过细（>30）——会增加题目数和诊断不确定性。建议每个学科核心属性 ≤ 20 个
- **与 MIRT 的关系**：CD-CAT 输出属性模式，MIRT 输出能力分——CTA 应**同时输出两种**（能力分给 LCA 决策，属性模式给 BloomProfile）

## 3. 核心理论 3：BKT / DKT（贝叶斯/深度知识追踪）

### 3.1 BKT 核心（Corbett & Anderson, 1994）

**BKT**（Bayesian Knowledge Tracing）把学生对每个知识点的掌握建模为 2 状态的隐马尔可夫模型：

```
状态：{未掌握, 已掌握}（二值）
参数：
  P(L₀) - 初始掌握概率（先验）
  P(T)  - 学习转移概率（未掌握→已掌握）
  P(G)  - 猜测概率（未掌握却答对）
  P(S)  - 失误概率（已掌握却答错）
```

更新规则：

```
P(L_n | 答对) = P(L_{n-1}) · (1 - P(S)) / [P(L_{n-1}) · (1 - P(S)) + (1 - P(L_{n-1})) · P(G)]
P(L_n | 答错) = P(L_{n-1}) · P(S) / [P(L_{n-1}) · P(S) + (1 - P(L_{n-1})) · (1 - P(G))]

P(L_n) = P(L_{n-1} | 观测) + (1 - P(L_{n-1} | 观测)) · P(T)
```

### 3.2 DKT 核心（Piech et al., 2015）

**DKT**（Deep Knowledge Tracing）用 RNN/LSTM 替代 BKT 的 2 状态 HMM：

```
输入：作答序列 (x₁, x₂, ..., x_t)  其中 x_t ∈ {0,1}ᴷ（K 个知识点）
模型：LSTM(h_{t-1}, x_t) → h_t → 输出层 → p_t ∈ [0,1]ᴷ
p_t[k] = 学生当前掌握知识点 k 的概率
```

**优势**：捕获跨知识点的关联（学生答对 a 提升 b 的掌握）。
**劣势**：黑箱、可解释性弱、需要大量数据。

**DKVMN**（Dynamic Key-Value Memory Networks, Zhang et al., 2017）是 DKT 的改进，引入动态键值记忆增强可解释性。

### 3.3 与 ECOS CTA 的对接

**BKT/DKT 是 CTA 5D 中 K 维度的时间演化引擎**：

| CTA 行为 | BKT/DKT 对应 |
|---|---|
| 学生做对一道"二次函数概念"题 | K(二次函数概念) 的 P(L) 上升 |
| 学生连续 3 道错 | K 对应 P(L) 下降 + BloomProfile 下降 |
| 学生一周未复习 | P(T) 衰减（与 §6 间隔效应整合）|

**BKT vs DKT 选择**：
- **BKT**：参数少、可解释、易调参、适合冷启动——**CTA MVP 阶段用 BKT**
- **DKT**：跨知识点关联强、可扩展、但需要数据——**CTA Phase 5+ 引入 DKT 作为对比基线**
- **DKVMN**：可解释的深度版本——**作为 Phase 6 研究目标**

### 3.4 借鉴决策

| 决策项 | 选择 | 理由 |
|---|---|---|
| **MVP 算法** | BKT（4 参数版本）| 简单、可解释、易工程化 |
| **每个知识点单独 BKT** | 是 | K 维度按知识点拆分（如"二次函数"、"分情况讨论"独立 BKT）|
| **跨知识点关联** | Phase 5+ 引入 DKT/DKVMN | MVP 不实现，避免复杂度爆炸 |
| **BKT 参数初始化** | P(L₀)=0.1, P(T)=0.1, P(G)=0.2, P(S)=0.1 | 通用先验，学科级调整 |
| **衰减** | 与"间隔效应"整合：未复习超过 N 天，P(L) 衰减 | 见 §6 POMDP |

### 3.5 实施注意事项

- **滑动窗口修正**：BKT 默认假设 P(T) 恒定，ECOS 应修正为"最近 N 次作答窗口内的学习率"
- **过度自信修正**：BKT 一旦 P(L) > 0.95 就稳定，但学生可能遗忘——ECOS 必须结合间隔效应衰减
- **冷启动优化**：新知识点 P(L₀) 不应是 0.1，应根据"前置知识点掌握度"推断（用知识图谱贝叶斯网络）

## 4. 核心理论 4：POMDP / Hidden Markov Models

### 4.1 核心观点

**POMDP**（Partially Observable Markov Decision Process）是强化学习的部分可观察版本：

```
POMDP = (S, A, O, T, R, Ω, γ)
  S - 状态空间（隐藏）
  A - 动作空间
  O - 观测空间（不完整）
  T - 状态转移 T(s'|s, a)
  R - 奖励 R(s, a)
  Ω - 观测概率 Ω(o|s')
  γ - 折扣因子
```

**Hidden Markov Model (HMM)** 是 POMDP 的无动作版本，仅有 T(s'|s) 和 Ω(o|s')。

**核心问题**：
- **信念更新（Belief Update）**：给定历史观测 (o₁, o₂, ..., o_t) 和动作 (a₁, ..., a_{t-1})，更新对隐藏状态的后验分布
- **信念状态** b(s) ∈ Δ(S)（在状态空间上的概率分布）

**信念更新公式**：

```
预测：b'(s') = Σ_s T(s'|s, a) · b(s)
更新：b(s') ∝ Ω(o | s') · b'(s')
```

### 4.2 与 ECOS CTA 的对接

**CTA 的 5D + BloomProfile 状态空间本质是一个 POMDP**：

| POMDP 元素 | ECOS CTA 对应 |
|---|---|
| 隐藏状态 S | (5D 状态, BloomProfile) 的连续/离散空间 |
| 动作 A | 干预类型（解释/练习/探究/反馈/苏格拉底）|
| 观测 O | 学生作答、答题时间、文本解释、反思日志 |
| 状态转移 T | 学习效应 + 遗忘衰减 + 干预影响 |
| 观测概率 Ω | 学生行为的随机性（猜测、失误、注意力波动）|
| 奖励 R | 状态改善度（如 P(L) 上升）|

**关键洞察**：CTA 不是简单地"用 BKT 更新 K 维度"，而是 **CTA 整体是一个 POMDP，5D 状态都是隐藏状态，作答是部分观测**。BKT/DKT 是 POMDP 在"单知识点二值状态"下的特例。

### 4.3 借鉴决策

| 决策项 | 选择 | 理由 |
|---|---|---|
| **状态空间** | 连续（MIRT 的 θ ∈ ℝ⁵）+ 离散（属性 α ∈ {0,1}ᴬ）混合 | 兼容 MIRT 的连续估计和 CD-CAT 的属性诊断 |
| **信念表示** | 多元高斯分布 N(μ, Σ)（连续）+ 离散分布（属性）| 统计性质好、可计算 |
| **更新算法** | 扩展卡尔曼滤波（EKF）+ 离散属性的精确推断 | 工程可行、性能可接受 |
| **LCA 集成** | LCA 选择动作 a，LCA 报告奖励 r，CTA 用 POMDP 更新信念 | CTA 与 LCA 通过 POMDP 接口解耦 |
| **长期规划** | Phase 5+ 用 POMCP（POMDP 的 MCTS 求解）| MVP 阶段用贪婪策略 |

### 4.4 实施注意事项

- **维度诅咒**：连续状态空间维度爆炸（5D + Bloom 6 层 = 11D），必须用因子化或稀疏表示
- **参数学习**：POMDP 的 T 和 Ω 参数需要从学生行为数据学习——MVP 阶段用合理先验 + 数据修正
- **可解释性**：CTA 的信念 b(s) 本身是统计对象——教师/家长报告应取 μ（均值）+ 不确定性（Σ 的迹），而非完整分布

## 5. 核心理论 5：Causal Inference（因果推断）

### 5.1 核心观点

**Causal Inference**（Pearl, 2009; Imbens & Rubin, 2015）是研究"干预 vs 观察"的数学框架。核心概念：

- **因果图（Causal DAG）**：节点 = 变量，边 = 因果方向
- **do-calculus**：表达"如果人为干预 X = x，结果会怎样"——区别于"观察到 X = x，结果会怎样"
- **反事实（Counterfactual）**：Y_x(u) = 在个体 u 上设置 X=x 的潜在结果
- **Average Treatment Effect (ATE)**：E[Y_{X=1} - Y_{X=0}]
- **Conditional ATE (CATE)**：在协变量 Z 下，A/B 实验组差异

**核心公式（Pearl's do-calculus）**：

```
P(Y = y | do(X = x)) = Σ_z P(Y = y | X = x, Z = z) · P(Z = z)
```

关键：**do() ≠ 观察**——do 是干预，强行切断 X 的因果上游。

### 5.2 与 ECOS CTA + LCA 的对接

**LCA 的干预效果归因本质是因果问题**——但当前 v2.0 §3.5 互校机制只提到"实验验证"，未明确因果框架。

**关键场景**：
- 学生在 LCA 干预 a 下进步了——是干预 a 的效果，还是自然成熟（maturation）？
- 学生 A 在干预 a 下进步，学生 B 在干预 b 下进步——是干预差异，还是学生个体差异？

**CTA 应承担因果归因职责**：

```
LCA 执行干预 a
    ↓
观察学生状态变化 Δs = s_t - s_{t-1}
    ↓
CTA 用因果推断判断：
  - 这是干预 a 的因果效果，还是 maturation/regression-to-mean？
  - 换干预 b 会怎样（counterfactual）？
    ↓
CTA 输出归因结果：
  - P(进步归因于干预 a) = ?
  - CATE：在 K 弱 P 强学生上，干预 a 比 b 优多少？
```

### 5.3 借鉴决策

| 决策项 | 选择 | 理由 |
|---|---|---|
| **核心框架** | Pearl 因果图（DoWhy 库）+ A/B 测试 | DoWhy 提供端到端因果推断 |
| **随机化单元** | 学生层 + 班级层（双层 A/B）| 个体效应 + 群体效应 |
| **协变量** | 5D 状态 + BloomProfile + LearningDNA | 高维协变量需要因果森林 |
| **估计方法** | Causal Forest（Wager & Athey, 2018）+ Doubly Robust Learner | 处理高维协变量 + 异质性 |
| **MVP 实施** | 简化版：单变量 A/B 测试 + T-test | Phase 4 验证效果量，Phase 5+ 升级到因果森林 |

### 5.4 实施注意事项

- **伦理边界**：教育干预的 A/B 测试涉及学生——必须获得家长/学校知情同意，不能"暗中"实验
- **基线对照**：必须有"无干预"对照组（自然成熟）——这是 ECOS 的伦理底线
- **效果量阈值**：教育干预的 effect size 通常 0.2-0.5（小到中），避免被噪声淹没
- **延迟效应**：干预效果可能延后显现（学习需要时间），CTA 必须支持 N 周后的延迟评估

## 6. 整合：CTA 信念分布的完整数学框架

5 个理论不是 5 个独立工具，而是 **CTA 信念分布的 5 层数学栈**：

```
┌─────────────────────────────────────────────────────────────────────┐
│ L4 因果归因层        │ Causal Inference（DoWhy + Causal Forest）   │
│                     │  → 干预效果归因 + 异质性处理                  │
├─────────────────────────────────────────────────────────────────────┤
│ L3 自适应选择层      │ CD-CAT（GDINA + PWKL 选题）                 │
│                     │  → 下一题/下一干预的自适应选择                │
├─────────────────────────────────────────────────────────────────────┤
│ L2 状态估计层        │ MIRT（5D 非补偿多维能力向量）                │
│                     │  → 5D 状态的联合估计 + Σ_θ 协方差             │
├─────────────────────────────────────────────────────────────────────┤
│ L1 时间演化层        │ BKT/DKT（单知识点掌握度演化）                │
│                     │  → P(L_n) 的更新（含间隔效应衰减）          │
├─────────────────────────────────────────────────────────────────────┤
│ L0 概率框架层        │ POMDP / HMM（隐藏状态 + 部分观测）          │
│                     │  → 信念状态 b(s) 的更新循环                  │
└─────────────────────────────────────────────────────────────────────┘
```

### 6.1 数据流（CTA 内部）

```
[学生作答 / 答题时间 / 反思文本]
        ↓ 感知层（LLM Critic）
[结构化观测 o_t]
        ↓
POMDP 框架预测：b'(s') = Σ_s T(s'|s, a) · b(s)
        ↓
MIRT 更新：μ_θ ← E-step/M-step（基于 o_t 的条件概率）
        ↓
BKT 更新：P(L_n | 观测) 更新（每个知识点独立）
        ↓
CD-CAT 选题：选下一题（信息量最大）
        ↓
输出信念分布 b(s) + BloomProfile + LearningDNA + Trajectory
        ↓
LCA 用 b(s) 选择干预 a_t
        ↓
[执行干预]
        ↓
等待下一观测 o_{t+1}（循环）
```

### 6.2 与 LLM 的关系

**关键澄清**：CTA **不是**用 LLM 直接生成 5D 状态估计，而是：

| 层级 | CTA 实现 | LLM 角色 |
|---|---|---|
| L0-L2 数学层 | 纯统计（MIRT/BKT/POMDP）| **零使用** LLM——保证数学严谨性 |
| 感知层（o_t 提取）| 结构化提取 | **LLM Critic** 把自然语言作答转结构化数据 |
| 解释层 | 模板 + 统计值 | **LLM** 生成自然语言解释（"学生在 K 维度较弱..."）|

**原则**：**数学层不调用 LLM**——这是 ECOS "科学化认知估计"的硬底线。任何让 LLM 直接生成信念估计的设计都是退路。

### 6.3 与 LCA 的接口

CTA 与 LCA 通过 POMDP 的 `(a_t, r_t, o_{t+1})` 接口协作：

```
CTA → LCA：b(s) 信念分布 + BloomProfile + LearningDNA
LCA → CTA：a_t 干预选择 + r_t 干预奖励
CTA → App：诊断报告 + 下一步建议
```

## 7. CTA 数学基础的 MVP 实施路线

| Phase | 内容 | 复杂度 |
|---|---|---|
| **MVP（Phase 4）**| BKT（4 参数）+ MIRT（5D）+ 简化 CD-CAT（GDINA 基础）| 中等（依赖成熟开源库：pyirt, pyBKT, GDINA）|
| **Phase 5** | POMDP 整合（LCA 决策统一接口）+ Causal Forest 归因 | 高（需要自研 POMCP 求解器）|
| **Phase 6** | DKT/DKVMN 跨知识点关联 + 完全 POMCP | 研究级 |

**关键开源依赖**：
- `pyBKT`（Python BKT 实现）
- `mirt`（R 包，Python 通过 rpy2 调用）
- `GDINA`（R 包，Cognitive Diagnosis）
- `DoWhy`（Microsoft 因果推断库）
- `pgmpy`（Python 概率图模型库）
