# 页面一：总览（认知架构与认知状态定义）

> **目标**：把人或其 Agent 的认知状态定义成可观测、可估计、可干预、可验证的状态集合，使学习系统能够诊断 A、指定 B，并设计从 A 到 B 的训练路径。

## 核心结论

认知状态不应定义为一个单一能力分数，而应定义为一个带置信度的多层状态向量：知识、程序技能、策略、注意/工作记忆资源、目标/动机、情绪负荷、元认知、世界模型、工具/Agent 外部记忆，以及上下文约束。A→B 学习系统的核心不是"讲授内容"，而是持续估计这些状态变量并选择能最大化状态迁移概率的干预。

## 当前工作定义

**认知状态** 是主体在某一任务族和情境下，能够解释其理解、选择、行动、错误、迁移和自我调节表现的一组潜变量及其证据状态。它不是主体完整心智的复制，而是为了预测和改变学习行为而构建的任务相关模型。

可操作表达：

```
S_t = {K, P, M, G, A, E, W, X, U}
```

其中 `K` 是概念/事实知识，`P` 是程序性技能，`M` 是元认知与策略，`G` 是目标/价值/意图，`A` 是注意与工作记忆状态，`E` 是情绪和动机负荷，`W` 是世界模型/信念，`X` 是外部工具、笔记、Agent 记忆和环境支架，`U` 是不确定性与证据质量。

## 为什么认知架构重要

ACT-R、Soar、LIDA、CLARION 等认知架构提供了"状态变量应该有哪些层"的启发：记忆、目标、规则/操作、注意、学习、动作选择和元认知。它们比单纯的成绩模型更适合定义人的可迁移认知状态。

## 为什么学习建模重要

知识追踪、IRT、认知诊断和智能导师系统提供了"如何从行为证据估计状态"的方法。没有可观测证据，认知状态定义会停留在哲学标签。

## 为什么 Agent 视角重要

语言 Agent 的 CoALA、Voyager 等工作把记忆、动作空间、反馈、自我验证和技能库工程化。它们提示：未来用户的认知状态可能分布在人、工具和个人 Agent 之间。

## 从 A 到 B 的最小系统闭环

| 阶段 | 系统问题 | 产物 | 关键风险 |
|------|---------|------|---------|
| 定义 B | 目标状态在任务中表现为什么？ | 能力规格、任务样本、评价量规、迁移场景 | B 被定义成课程完成度，而不是可迁移能力 |
| 估计 A | 当前证据支持哪些状态判断？ | 状态向量、置信度、缺失证据清单 | 把一次测试结果误当稳定认知状态 |
| 规划路径 | 哪些干预最可能改变目标变量？ | 练习序列、反馈策略、支架、反思任务 | 只补知识，不改变策略、动机和情境使用 |
| 执行更新 | 学习中状态如何变化？ | 轨迹、错误模式、遗忘/迁移信号 | LLM 对学习者状态的判断不稳定或不可校准 |
| 验证 B | 是否在新任务、新情境中达到目标？ | 迁移测评、延迟测评、解释性报告 | 只验证训练集任务，未验证泛化 |

## 关键新增内容

- 建立了"认知状态向量"初版，连接认知架构、学习科学和 Agent 工程。
- 形成相关工作矩阵，比较经典认知架构、学习者建模、语言 Agent 架构。
- 提出从 A 到 B 的学习系统闭环：目标定义、诊断、干预、状态更新、迁移验证。
- 标注了证据类型与不确定性，避免把工程假设误写成已证实事实。

# 页面二：背景

## 一行定义

认知架构是对智能主体如何感知、记忆、推理、学习、选择行动和自我调节的计算性组织方式；认知状态则是该架构在某一时刻、某一任务族中的参数化实例。

## 关键区分

| 概念 | 本研究中的含义 | 容易混淆之处 |
|------|--------------|-------------|
| 认知架构 | 描述心智/Agent 的模块、记忆、控制流、学习机制和行动选择。 | 不是某个单一 AI 模型，也不等同于神经网络结构。 |
| 认知状态 | 任务相关的潜变量集合，用来解释和预测主体表现，并指导干预。 | 不是完整人格、意识状态或脑状态的全量扫描。 |
| 学习状态 | 认知状态中与可训练能力变化相关的子集，包括知识掌握、策略、动机、负荷和迁移准备度。 | 不是课程进度条或答题正确率。 |
| Agent 认知状态 | Agent 的记忆、工具、目标、计划、约束、技能库、信念和不确定性。 | 不等同于 LLM 当前上下文窗口；长期记忆和外部环境同样重要。 |

## 认知状态的基本层

**知识层**

事实、概念、关系、范畴、例子和误概念。常用证据包括测试题、解释质量、概念图和迁移题。

**程序层**

完成任务的步骤、规则、技能、启发式和自动化程度。ACT-R 与认知导师系统尤其强调这一层。

**控制层**

目标选择、注意分配、工作记忆负荷、计划、监控和错误修正。Soar 的操作符选择和 LIDA 的认知循环提供启发。

**动机/情绪层**

兴趣、价值、压力、自我效能、疲劳和挫败感。它们会影响练习持续性、探索范围和反馈接受度。

**元认知层**

主体对自己理解程度、策略有效性、错误原因和下一步学习计划的判断。

**外部支架层**

笔记、搜索、工具、个人 Agent、环境线索和社交协作。未来认知状态会越来越多地分布在人体与工具之间。

## 边界条件

- 定义认知状态必须绑定任务族，否则状态维度会无限膨胀且不可验证。
- 状态必须带证据和置信度；没有观测渠道的维度只能作为假设。
- 学习目标 B 必须可表现、可迁移、可延迟验证，而不是只在训练题上有效。
- 人和 Agent 的状态可以共享抽象层，但底层机制不同，不能直接等同。

## 优先知识块

1. 经典认知架构：ACT-R、Soar、CLARION、LIDA。
2. 学习者建模：知识追踪、IRT、认知诊断、智能导师系统。
3. 状态估计：潜变量、贝叶斯更新、序列模型、不确定性校准。
4. 干预设计：练习、反馈、脚手架、反思、间隔重复、迁移任务。
5. Agent 工程：记忆、计划、工具调用、技能库、反馈循环、自我验证。

# 页面三：相关工作

## 相关工作矩阵

| 工作/传统 | 核心主张 | 状态变量启发 | 擅长解释 | 不足 | 对 A→B 系统的价值 |
|----------|---------|------------|---------|------|-----------------|
| ACT-R | 用模块、缓冲区、声明性/程序性知识和生产规则解释人类认知。 | 事实块、规则、激活、检索、目标缓冲、感知/运动模块。 | 技能获得、反应时、错误模式、认知导师建模。 | 工程应用需要精细任务分析；对情绪、社会和开放式任务支持较弱。 | 适合把 B 定义为专家规则和程序性技能，并用 model tracing 跟踪 A。 |
| Soar | 通用智能可由状态、操作符、问题空间、记忆和学习机制组织。 | 工作记忆、操作符偏好、目标、语义记忆、情景记忆、强化学习、chunking。 | 问题求解、规划、交互式任务学习、机器人/游戏 Agent。 | 符号建模成本高；人类心理细节不如 ACT-R 精细。 | 适合把 A→B 看成问题空间搜索和操作符学习。 |
| CLARION | 显性/隐性过程并行互动，认知包含动作、非动作、动机和元认知子系统。 | 显性规则、隐性联结、动机强度、元认知控制。 | 技能学习中"会做但说不清"和"规则指导练习"的互动。 | 实际部署和工具生态不如 ACT-R/Soar 普及。 | 提醒认知状态不能只记录可陈述知识，还要记录隐性技能和动机。 |
| LIDA / Global Workspace | 认知由循环的理解、注意/意识广播、行动选择和学习组成。 | 感知、情景模型、注意竞争、全局广播、行为选择、记忆更新。 | 注意、意识、行动选择、多记忆系统协同。 | 对教育测量和学习路径优化的直接方法较少。 | 适合定义学习中的"当前情境模型"和注意瓶颈。 |
| 智能导师系统 | 学习系统通常包含领域模型、学生模型、教学模型和界面模型。 | 领域规则、学生掌握状态、错误诊断、教学策略。 | 个性化反馈、步骤级指导、技能掌握跟踪。 | 依赖领域建模；开放式能力和长期目标建模困难。 | 提供从状态估计到干预选择的工程闭环。 |
| BKT / DKT / 知识追踪 | 从交互序列估计学习者知识掌握随时间变化。 | 每个技能的掌握概率、学习率、遗忘、猜测、失误、序列隐状态。 | 题目作答预测、掌握度估计、练习序列优化。 | 容易过窄地把认知状态压缩成题目正确率；可解释性与泛化需谨慎。 | 是估计 A 和监测 A→B 轨迹的基础方法。 |
| IRT / 认知诊断 | 把表现视为潜在能力/属性与题目参数的函数。 | 能力值、题目难度、区分度、属性掌握向量、Q-matrix。 | 测评设计、题库校准、能力估计、诊断报告。 | 强依赖测评设计；难以覆盖复杂任务中的过程策略。 | 帮助把 B 变成可测量标准，并估计测量误差。 |
| BDI Agent | Agent 行为可由信念、欲望/目标和意图/承诺组织。 | 信念库、目标、意图、计划库、事件队列。 | 目标驱动行为、计划选择和执行承诺。 | 传统 BDI 本身学习机制较弱。 | 适合定义个人 Agent 的目标、计划和信念状态。 |
| CoALA 语言 Agent | 语言 Agent 可用模块化记忆、结构化动作空间和决策过程组织。 | 工作/情景/语义/程序记忆，内外部动作，决策循环。 | 现代 LLM Agent 的系统分类与设计方向。 | 多为工程框架，人的学习状态仍需教育测量支持。 | 连接人的认知状态与个人 Agent 状态的候选桥梁。 |
| Voyager 等开放式 Agent | 自动课程、技能库、环境反馈和自我验证可支持开放式终身学习。 | 探索目标、代码技能库、反馈历史、错误修正、任务进展。 | Agent 技能积累、复用和开放环境探索。 | 主要是具身游戏环境；迁移到人类学习需要重新定义证据和伦理边界。 | 启发个人 Agent 如何作为外部认知支架参与 A→B。 |

## 谱系概览

| 时期 | 主要关注 | 对本项目的遗产 |
|------|---------|--------------|
| 1950s-1970s | 问题求解、符号 AI、认知心理学、生产系统。 | 把认知看成可计算的信息加工过程。 |
| 1980s-1990s | 统一认知理论、ACT-R、Soar、智能导师系统。 | 把知识、规则、目标和学习机制工程化。 |
| 2000s-2010s | 大规模认知架构综述、学习数据挖掘、知识追踪、认知诊断。 | 把状态估计与真实学习系统数据连接起来。 |
| 2020s | LLM、语言 Agent、记忆增强、工具使用、混合智能导师。 | 把认知状态扩展到人-工具-Agent 分布式系统。 |

# 页面四：综合框架

## 框架 1：认知状态向量

| 维度 | 含义 | 可观测证据 | 可能估计方法 | 干预杠杆 |
|------|------|----------|------------|---------|
| K 知识 | 事实、概念、关系、误概念。 | 解释、选择题、概念图、问答、迁移题。 | IRT、认知诊断、概念图评分、LLM rubric 评估加人工校准。 | 例子、反例、类比、检索练习、概念冲突。 |
| P 程序技能 | 步骤、规则、操作符、自动化程度。 | 解题轨迹、时间、错误类型、步骤选择。 | BKT/DKT、model tracing、过程挖掘。 | 分步练习、即时反馈、变式训练、刻意练习。 |
| M 元认知/策略 | 计划、监控、调试、策略选择和自我解释。 | 学习日志、反思、提示使用、错误修正行为。 | 行为序列分类、文本分析、教师/Agent 评分。 | 自我解释、预测-验证、错因归因、策略对比。 |
| G 目标/价值/意图 | 正在追求什么、为何追求、承诺强度。 | 目标陈述、选择行为、放弃点、任务切换。 | 偏好学习、BDI 风格目标模型、问卷与行为混合估计。 | 目标重构、价值连接、任务分解、承诺机制。 |
| A 注意/工作记忆 | 当前负荷、焦点、可用资源和干扰。 | 反应时、停顿、回看、遗漏、主观负荷。 | 认知负荷估计、眼动/交互日志、表现波动模型。 | 脚手架、分块、外化、节奏控制、提示。 |
| E 情绪/动机负荷 | 焦虑、挫败、兴趣、自我效能、疲劳。 | 自评、语气、坚持时间、帮助请求、退出行为。 | 多模态情感识别、问卷、事件触发规则。 | 难度调节、反馈语气、成功经验、恢复时间。 |
| W 世界模型/信念 | 主体对任务、因果、约束和自我能力的信念。 | 解释、预测、计划、错误假设。 | 信念图谱、因果模型、自然语言声明抽取。 | 反事实案例、模拟、澄清问题、信念修正。 |
| X 外部支架 | 笔记、工具、搜索、个人 Agent、环境和社交资源。 | 工具调用、笔记质量、Agent 记忆、协作记录。 | 知识库审计、工具使用分析、Agent memory tracing。 | 笔记结构、工具训练、Agent 提示、环境设计。 |
| U 不确定性 | 每个状态判断的置信度、证据来源和冲突。 | 测量误差、证据覆盖、模型分歧。 | 贝叶斯后验、置信区间、校准曲线、证据图。 | 追加诊断、主动测评、任务抽样、人工复核。 |

## 框架 2：A→B 迁移模型

把学习看成带约束的状态转移问题：

```
S_{t+1} = update(S_t, observation_t, intervention_t, context_t)
policy(S_t, B) -> intervention_t
```

系统目标不是一次性判断"会/不会"，而是选择干预，使目标状态 B 的后验概率随时间增加，并在新情境中保持。

| 对象 | 需要定义 | 示例 |
|------|---------|------|
| A | 初始状态及置信度。 | 概念 K2 掌握概率低；程序 P4 可在提示下完成；元认知 M1 弱；负荷 A 高。 |
| B | 目标状态的表现标准。 | 能在新问题中选择正确策略，解释理由，发现错误并修正，延迟一周仍能迁移。 |
| 路径 | 可用干预及成本。 | 诊断题、例题对比、分步练习、反馈、反思、项目任务、Agent 辅助。 |
| 验证 | B 是否真实达到。 | 不同表面特征的新题、真实任务、口头解释、无提示完成、长期保持。 |

## 框架 3：人-Agent 分布式认知状态

未来"一个人或他的 Agent 的认知状态"不应只看人的脑内变量，也要看个人 Agent 的外部记忆、工具技能和协作协议。一个用户可能"不记得细节"，但其 Agent 拥有可靠笔记、代码、计划和检索能力，因此系统能力状态高于裸人状态。

| 层 | 人的状态 | Agent 的状态 | 联合状态问题 |
|---|---------|------------|------------|
| 记忆 | 长期记忆、工作记忆、情景回忆。 | 向量库、结构化笔记、日志、偏好。 | 哪些知识必须内化，哪些可以可靠外包？ |
| 技能 | 熟练操作、判断、解释、迁移。 | 工具调用、代码技能、工作流模板。 | 人需要学会判断，Agent 可以承担重复执行。 |
| 目标 | 价值、意图、兴趣、约束。 | 任务队列、计划、优先级、提醒。 | Agent 是否真实反映人的目标，还是放大短期偏好？ |
| 证据 | 表现、解释、自评、行为轨迹。 | 调用日志、输出质量、错误修复、检索命中。 | 如何区分人的掌握与 Agent 的代偿？ |

## 系统原型建议

1. 从一个具体领域开始，例如"阅读并复现一篇机器学习论文"或"掌握概率图模型基础"。
2. 建立 B 的任务库：基础题、解释题、迁移题、项目任务、延迟测评。
3. 为每个任务标注状态维度：K/P/M/G/A/E/W/X。
4. 用 BKT/IRT/认知诊断估计窄技能，用 LLM rubric 辅助评估开放式解释，但保留校准集。
5. 把个人 Agent 的记忆和工具能力作为 X 维度单独记录，避免混淆"人学会了"和"Agent 帮会了"。
6. 持续记录每次干预后的状态更新，比较哪类干预最能推动目标状态。

# 页面五：假设与问题

## 核心假设

| 假设 | 为什么可能成立 | 支持证据 | 削弱/证伪证据 | 如何测试 | 置信度 |
|------|--------------|---------|--------------|---------|-------|
| H1：多层认知状态比单一掌握度更能预测迁移。 | 迁移依赖知识、策略、动机、负荷和情境支架，不只是题目正确率。 | 认知架构均包含多模块；ITS 文献强调学生模型不仅是知识点。 | 如果单一技能掌握概率在多领域稳定优于多层模型。 | 对比单一 BKT/DKT、知识+策略模型、完整状态模型对迁移题和延迟题的预测。 | B/C |
| H2：个人 Agent 的状态必须作为外部支架 X 单独建模。 | Agent 记忆和工具会改变用户可完成任务，但不等于人的内化能力。 | CoALA 和 Voyager 等语言 Agent 工作显示外部记忆、技能库和反馈循环能显著改变 Agent 能力。 | 如果有证据表明 Agent 代偿不会影响学习诊断或迁移表现。 | 设计有/无 Agent 支架、可见/不可见笔记条件，比较人的独立表现与联合表现。 | C |
| H3：LLM 可以辅助开放式状态解释，但不能单独承担高风险学习者建模。 | LLM 擅长语言评价，但状态更新需要时序一致、校准和可解释证据。 | 2025 arXiv 研究报告 DKT 在 next-step prediction 和时序一致性上优于测试中的 LLM；这是预印本证据。 | 如果后续大规模研究显示 LLM 在校准、时序一致和早期诊断上稳定优于专门模型。 | 用真实学习日志比较 LLM rubric、DKT/BKT、混合模型的 AUC、校准误差、更新方向错误率。 | B/C |
| H4：B 的定义应优先用任务族与迁移标准，而非知识点清单。 | 真实认知能力表现为在变形任务中选择策略、解释和修正。 | 认知导师和架构传统都强调过程、规则和任务表现。 | 如果知识点清单能稳定预测复杂迁移表现。 | 比较"知识点掌握 B"和"任务族能力 B"对真实项目表现的预测。 | B |

## 开放问题树

| 根问题 | 子问题 | 需要的证据 | 优先级 |
|-------|--------|----------|-------|
| 认知状态的最小充分维度是什么？ | 哪些维度在不同领域稳定有预测力？哪些只是领域特异？ | 跨领域学习数据、消融实验、预测迁移任务的模型比较。 | 高 |
| 如何定义 B？ | B 是专家状态、可用能力、任务通过标准，还是长期发展方向？ | 目标任务分析、专家表现样本、学习者迁移表现。 | 高 |
| 如何避免错误诊断？ | 一次失败是知识缺失、注意波动、动机问题还是题目歧义？ | 多证据测量、模型置信区间、主动诊断任务。 | 高 |
| 人的学习和 Agent 代偿如何区分？ | 哪些任务必须无 Agent 测评？哪些任务应评价人-Agent 联合能力？ | 有无工具条件下的表现差异、解释质量、延迟迁移。 | 中高 |
| 状态干预策略如何优化？ | 给定 A 和 B，何时讲解、练习、提问、反馈、反思或交给 Agent？ | 随机对照、bandit/RL 策略比较、长期保持数据。 | 中高 |

## 风险登记

- **构念污染**：把"会用 Agent"误判为"人已掌握"。应区分独立能力和联合能力。
- **状态过拟合**：模型只预测训练平台题目，无法预测真实任务。
- **黑箱诊断**：LLM 给出流畅解释但缺乏校准证据。
- **伦理风险**：情绪、动机和价值状态属于敏感建模对象，需要最小化采集、可解释和用户控制。
- **目标错置**：B 若由系统单方面定义，可能优化短期绩效而损害用户长期自主性。

## 下一步实验原型

建议选择一个窄领域先做 2-4 周原型：定义 20-40 个任务，覆盖知识题、步骤题、解释题、迁移题和项目题。记录用户作答、解释、提示使用、Agent 使用和反思文本。用 BKT/IRT 估计窄技能，用 rubric 评估开放式能力，用人工抽样校准 LLM 评分。最后比较三种模型对迁移任务的预测：单一分数、多技能模型、多层认知状态模型。

## 重要不确定性

- **维度最小性**：K/P/M/G/A/E/W/X/U 是综合框架，不是经实验确认的最小充分集。
- **跨领域泛化**：数学、编程、科研阅读、管理决策的认知状态维度权重可能不同。
- **LLM 评估可靠性**：LLM 可评估开放式解释，但必须与人工评分、行为数据和校准集结合。
- **人-Agent 边界**：联合认知能力如何评价仍是开放问题，尤其涉及责任、隐私和自主性。
