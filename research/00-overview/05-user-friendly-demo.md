## 1. ECOS 在做什么：一句话核心

### 1.1 思想方针：抽象-具象的循环

ECOS 的核心思想，可以用一句话概括：

> **ECOS 把成千上万的具象化学习过程（做题、答题、反思），进行认知状态抽象，然后分析抽象数据研究转换策略，再映射回具象化行动，如此循环不断演进。**

这段话听起来抽象，但我们来拆解它的每一步：

```
具体行为层（做题、答题、反思）
        ↕ 抽象化
认知状态层（学生在各个维度上的"心里分数"）
        ↕ 策略化
干预策略层（AI 教练选择的教学方法）
        ↕ 具象化
具体干预层（讲解视频、变式练习、引导问题）
        ↓
（回到第一步——新的具体行为）
```

这四个层次不断循环，每次循环都让学生的状态往目标方向移动一点点。

### 1.2 三个关键词

| 关键词 | 通俗版 | 专业版 | 比喻 |
|---|---|---|---|
| **理解学生** | AI 心理学家（CTA） | Cognitive Twin Agent——认知孪生 Agent | 体检医生，告诉你身体各个指标 |
| **改变学生** | AI 教练（LCA） | Learning Coach Agent——学习教练 Agent | 私人教练，制定训练计划 |
| **长期陪伴** | 6-12 年的成长档案 | 跨年级/跨学科的认知轨迹追踪 | 从小学到高中都有档案 |

### 1.3 互校循环：两位 AI 如何互相"挑刺"

ECOS 最特别的设计是**两位 AI 会互相"挑刺"**——这不是为了让系统更聪明，而是**防止 AI 犯大错误**。

```
┌─────────────────────────────────────────────────────────────┐
│                  互校循环（Calibration Loop）                    │
│                                                             │
│   CTA（AI 心理学家）提出假设                                  │
│       ↓                                                      │
│   "这个学生可能理解了分数概念，但我不太确定，再观察一次"            │
│       ↓                                                      │
│   LCA（AI 教练）设计实验                                      │
│       ↓                                                      │
│   "好，我给他出一道变式题，测试他是不是真的懂了"                  │
│       ↓                                                      │
│   学生做题（观察结果）                                         │
│       ↓                                                      │
│   CTA 更新信念                                               │
│       ↓                                                      │
│   "这道题他用错了方法——原来他的理解是错的！更新：他没有真正理解"    │
│       ↓                                                      │
│   LCA 重新规划                                               │
│       ↓                                                      │
│   "既然他误解了，那我换一个方法——用披萨比喻重新讲"              │
│       ↓                                                      │
│   （回到 CTA 提出新假设）                                      │
└─────────────────────────────────────────────────────────────┘
```

**为什么需要两位 AI？**

- 如果只有一位 AI：它说什么就是什么，没有制衡——容易"幻觉"（一本正经地胡说八道）
- 有两位 AI 互相制衡：CTA 提出假设，LCA 用实验验证，CTA 再更新——像科学实验一样，有假设、有验证、有修正

## 2. 学生小明是谁：A 状态（起点）

### 2.1 基础信息

| 维度 | 信息 |
|---|---|
| **姓名** | 小明 |
| **年龄 / 年级** | 9 岁 / 小学四年级 |
| **学科** | 数学 |
| **当前问题** | 分数计算总出错 |
| **家长** | 已签署数据使用同意书 |

### 2.2 小明现在的真实状态（A 状态快照）

**5 个维度的"心里的分数"**（通俗版）：

| 维度 | 含义 | 小明的分数 | 通俗描述 |
|---|---|---|---|
| **K（知识）** | 知道什么 | 0.55 | 基本概念听过，但容易混淆 |
| **P（程序）** | 会做什么步骤 | 0.35 | 计算速度慢，容易算错 |
| **S（策略）** | 会不会想办法 | 0.45 | 有时候会想，但不太主动 |
| **C（置信度）** | 自己觉得怎么样 | **0.75** | ★ 伪自信——觉得自己挺好的，实际一般 |
| **X（外部支架）** | 需要多少帮助 | 0.60 | 需要家长辅导 |

**认知的六个台阶（BloomProfile）**：

```
L1 记忆（记得公式）   ████████░░  80%  — 基本记得
L2 理解（懂意思）     ██████░░░░  60%  — 大概懂
L3 应用（用到新题）   ███░░░░░░░  30%  ★ 核心弱点
L4 分析（拆解问题）   ██░░░░░░░░  20%  — 几乎不会
L5 评价               ░░░░░░░░░░   5%  — 不涉及
L6 创造               ░░░░░░░░░░   0%  — 不涉及
当前主要在 L3（应用）——能记住公式，但用到新题就蒙了
```

**关键转折点（TC）状态**：

小明正在跨越的门槛：
- **TC "分数"**：前 TC 状态（还没真正理解"部分与整体"的关系）

**常见误解（Misconception）清单**：

小明可能有以下误解：
- ❌ "分母大 → 分数大"（例：以为 1/8 比 1/5 大）
- ❌ "分子大 → 分数大"（例：以为 3/4 比 2/3 大）
- ❌ 分数相加 = 分子分母各自相加（例：1/2 + 1/3 = 2/5）

### 2.3 一个具体的错误

小明做了一道这样的题：

> **题目**：比较 1/2 和 1/8 哪个更大？

**小明的答案**：1/8 更大（分母 8 > 分母 2）

**为什么会错**：

小明心里的"分数"是混乱的——他觉得"分母大"意味着"份数多"，所以 1/8 > 1/2。

**这道错题透露了什么**：
- 他的分数知识（K）是模糊的
- 他的程序技能（P)也没有建立"通分"等标准化步骤
- 他的置信度（C）是虚高的——他答完觉得自己对

### 2.4 家长和老师的常见困惑

**家长的困惑**：
> "小明分数题总错，但我看他课本内容都懂了呀？到底哪里出了问题？"

**老师的困惑**：
> "小明不是不理解分数，他只是粗心——但为什么每次都粗心？"

**ECOS 的回答**：
> "小明不是'粗心'，而是有一个系统的误解（Misconception）：他以为'分母大→分数大'。这不是记不记得的问题，而是认知结构需要重建。"

## 3. ECOS 如何"看见"小明的状态

### 3.1 三个"AI 角色"介绍

#### 角色 1：CTA（AI 心理学家）——Cognitive Twin Agent

**通俗版描述**：CTA 像一个"认知心理学家 + 心理测量学家"的结合体。它保守、严谨、有证据才下结论。

**它做什么**：
- 收集学生的行为数据（做题记录、答题时间、解释文本）
- 用科学方法估计学生在各个维度上的"心里的分数"
- 维护一个置信度——如果证据不足，它会说"我不确定"
- 检测学生的常见误解（Misconception）

**它不做什么**：
- ❌ 不会看一眼学生就说"你数学不好"（没有证据不下结论）
- ❌ 不会被学生的自我评价误导（学生常说"我觉得我懂了"但实际没有）
- ❌ 不会让 LLM 直接估计（LLM 会幻觉，必须用贝叶斯等科学方法）

**CTA 的思维模式**：认知科学家 + 心理测量学家——**保守、基于证据、维护置信度、避免幻觉**

#### 角色 2：LCA（AI 教练）——Learning Coach Agent

**通俗版描述**：LCA 像一个"私人教练 + 强化学习策略器"的结合体。它主动、探索、优化，不断尝试更好的方法。

**它做什么**：
- 基于 CTA 的状态估计，选择下一步应该怎么教
- 从多种教学方法中选择最可能有效的一个
- 设计"实验"——出一道题测试学生的真实理解
- 追踪效果，积累经验，持续优化策略

**它不做什么**：
- ❌ 不会按固定顺序讲知识（不是照本宣科）
- ❌ 不会一次讲太多内容（懂认知负荷理论）
- ❌ 不会只推荐难题或只推荐简单题（懂"跳一跳够得着"的原理）

**LCA 的思维模式**：教练 + 强化学习策略器——**主动、实验、探索、优化**

#### 角色 3：互校循环（CTA + LCA 的协作机制）

**通俗版描述**：两位 AI 会互相"挑刺"——CTA 提出假设，LCA 设计实验验证，CTA 再更新，形成闭环。

```
┌─────────────────────────────────────────────────────────────┐
│                  互校循环（完整版）                              │
│                                                             │
│   【第一步】CTA 提出假设                                       │
│   "根据小明做错的这道题，我估计他对'分数大小比较'的理解有问题"    │
│   置信度：0.70（中等确定）                                    │
│                                                             │
│   【第二步】LCA 设计实验                                      │
│   "好，我给他出三道变式题，专门测试分数大小比较"               │
│   干预类型：诊断型练习（Diagnostic Practice）                  │
│                                                             │
│   【第三步】学生做题（观察结果）                                 │
│   小明做了三道题，错了一道                                      │
│                                                             │
│   【第四步】CTA 更新信念                                       │
│   "三道题对两道，错了一道——我之前的假设部分正确                 │
│   但他不是完全不懂，他懂 2/3"                                 │
│   置信度提升到 0.85（更有把握了）                              │
│                                                             │
│   【第五步】LCA 重新规划                                       │
│   "既然他部分理解，那下一步不是重新讲概念，                       │
│   而是给他针对性练习——'分母大→分数小'这个规则要专门练"         │
│   干预类型：变式练习（Varied Practice）                        │
└─────────────────────────────────────────────────────────────┘
```

#### 部分 2：认知的六个台阶

用"攀登阶梯"来比喻：

```
              登顶
             ╱    ╲
           L6    L5   （创造 / 评价）——小学阶段基本不涉及
          ╱

       ╱
```

#### 部分 2.5：认知台阶的两层结构（BloomProfile）

ECOS 的认知台阶（BloomProfile）实际上是**两层结构**：

| 层 | 粒度 | 通俗比喻 | 小明的例子 |
|---|---|---|---|
| **学生级** | 6 层分布（聚合）| 全班平均成绩单 | 整体在 L3 应用层偏弱 |
| **知识点级** | 知识点 × 6 层 | 各科成绩单 | 分数 L3=0.30，但整数加法 L3=0.65 |

**为什么要分两层？**

```
【学生级 BloomProfile】
小明整体在 L3 应用层：0.30（弱）
——这个数字告诉你"小明应用能力整体偏弱"

【知识点级 BloomGoal】（小明的例子）
├── 分数 L3 应用：0.30  ★ 核心弱点（需要重点补）
├── 整数加法 L3 应用：0.65  （还可以）
├── 整数减法 L3 应用：0.60  （还行）
├── 乘法口诀 L3 应用：0.80  （很好）
└── 图形认识 L3 应用：0.55  （中等）

——这个详细分解告诉你"具体哪里弱、哪里强"
```

**为什么不能只用一个？**

- 只看学生级：只知道"整体弱"，不知道"哪里弱"
- 只看知识点级：失去整体判断，无法说"这个学生整体处于什么水平"
- 两层结合：既能说"整体 L3"，又能说"分数 L3 只有 0.30"

> **小明的实际数据**：
> - 学生级 BloomProfile：L1=0.80, L2=0.60, L3=0.30, L4=0.20
> - 知识点级 BloomGoal（分数）：L1=0.85, L2=0.65, L3=0.30, L4=0.15

### 4.2 Day 3：第一次互校循环

**场景**：小明做了一道测试题，系统发现了他对自己分数的判断有问题。

**小明的做题过程**：

> **题目**：比较 1/4 和 1/6 哪个更大？

**小明的答案**：1/4 更大（因为 4 < 6，所以 1/4 > 1/6）——**正确 ✓**

**但是！** LLM 在评分时发现了异常：

## 6. ECOS 在做什么：抽象-具象的递归循环

### 6.1 四层循环详解

ECOS 的核心是**四层循环**，每层都有具体含义：

```
层 1：具体行为层
═══════════════════════════════════
学生做的具体事情：
- 做一道分数比较题（具体行为）
- 花了 3 分钟（具体行为）
- 画了一个圆（具体行为）
- 说"我觉得我对了"（具体行为）

层 2：认知状态层（CTA 维护）
═══════════════════════════════════
从具体行为推断的抽象状态：
- K（知识）：0.55 → 0.65（概念理解有进步）
- P（程序）：0.35 → 0.52（步骤更熟练）
- C（置信度）：0.75 → 0.62（伪自信被纠正）
- Misconception：M-F2 命中 → M-F2 已部分纠正
- TC 状态：前 TC → liminal → 接近 post_liminal

层 3：干预策略层（LCA 维护）
═══════════════════════════════════
从抽象状态选择的抽象策略：
- 干预类型：从"讲解型"切换到"变式练习"
- CLT 级别：从 NOVICE 降到 DEVELOPING（减少 scaffolding）
- Bloom 目标：保持在 L3 应用，稳固后再向 L4 推进

层 4：具体干预层（App 层执行）
═══════════════════════════════════
落到具体的学习内容：
- 推送一道变式题（具体内容）
- 配一张披萨图解（具体形式）
- 要求学生自己解释思路（具体任务）
```

### 6.2 循环的递归性

**重要**：这个循环不是"跑一次就结束"，而是**递归的**——每完成一轮，新状态成为下一轮的起点：

```
第 1 轮：A0 ──→ A1
第 2 轮：A1 ──→ A2
第 3 轮：A2 ──→ A3
...
第 6 轮：A5 ──→ A6 = B（目标达成）
```

每次循环：
- 状态在微调（不是跳跃）
- 策略在适应（根据反馈调整）
- 误解在清除（一个一个纠正）
- TC 在推进（从"前"到"后"）

### 6.3 四层记忆：数据如何存储

ECOS 不仅"理解"和"改变"学生，还**记住**一切——从单次做题到跨学期的成长轨迹。ECOS 的记忆系统分为**四层**，每层有不同的"记忆时间"和"用途"：

```
┌─────────────────────────────────────────────────────────────────┐
│                     ECOS 四层记忆系统                               │
│                                                                 │
│  【第一层】短期记忆（Hawking 风格）                                  │
│  ───────────────────────────────────────────────────────────    │
│  存在哪：内存中（电脑/手机运行内存）                               │
│  记什么：最近 10-20 次做题记录 + 当前状态缓存                      │
│  生命周期：一次会话（今天打开用到关闭）                            │
│  比喻：大脑的"工作记忆"——正在想的事情                              │
│  小明的例子：今天做了 5 道题，这 5 道题的细节在短期记忆里            │
│                                                                 │
│                          ↓ 写入                                   │
│                                                                 │
│  【第二层】中期记忆（Crystallizer 风格）                            │
│  ───────────────────────────────────────────────────────────    │
│  存在哪：数据库主表                                               │
│  记什么：最近 4 个月的干预效果 + BloomProfile 变化快照             │
│  生命周期：一个学期（4 个月）                                      │
│  比喻：大脑的"近期记忆"——最近学的重要事情                          │
│  小明的例子：这学期做了 100 道题，TC "分数"从"前"到"后"的转变记录   │
│                                                                 │
│                          ↓ 持久化                                 │
│                                                                 │
│  【第三层】长期记忆（Identity 风格）                                 │
│  ───────────────────────────────────────────────────────────    │
│  存在哪：数据库主表（加密存储）                                    │
│  记什么：学生 5D 核心状态 + BloomProfile + LearningDNA           │
│  生命周期：6-12 年（从小学到高中毕业）                             │
│  比喻：大脑的"长期记忆"——记住你是谁、你会什么                       │
│  小明的例子：从四年级到六年级，每年分数水平的变化曲线                 │
│                                                                 │
│                          ↓ 归档                                   │
│                                                                 │
│  【第四层】永久记忆（Archive 风格）                                 │
│  ───────────────────────────────────────────────────────────    │
│  存在哪：数据库归档表（不可修改）                                   │
│  记什么：跨学期的成长轨迹快照 + 跨学段画像变化                      │
│  生命周期：永久（用于回顾、分析、研究）                             │
│  比喻：大脑的"永久档案"——重要经历永远保存                          │
│  小明的例子：小学毕业时的数学能力评估报告                            │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**为什么要分四层？**

| 维度 | 理由 |
|---|---|
| **数据量** | 每个学生每天产生几十条记录，四层分离避免数据爆炸 |
| **访问速度** | 短期记忆最快（内存），长期记忆次之，归档最慢（但查询少）|
| **隐私保护** | 近期数据家长可查阅，历史归档仅研究者可用 |
| **系统稳定** | 短期记忆丢了可以重建，永久归档丢了就真的丢了 |

**小明的例子**：

```
小明今天做了 5 道分数题：
→ 5 道题的细节（短期记忆）→ 今晚存到数据库
    → 今天的干预效果（中期记忆）→ 一个学期后汇总
        → 这学期的 5D 状态快照（长期记忆）→ 保持到小学毕业
            → 小学毕业的成长报告（永久记忆）→ 永久归档
```

## 7. 这套系统为什么不一样

### 7.1 vs 传统家教

| 维度 | 传统家教 | ECOS |
|---|---|---|
| **认知状态理解** | 凭经验判断（"这孩子粗心"）| 有证据的多维画像 |
| **教学方法** | 固定讲法（讲完练完）| 根据状态动态调整 |
| **误解检测** | 难发现（"他就是马虎"）| 有系统性误解库 |
| **长期追踪** | 每次课重新认识 | 6-12 年成长档案 |
| **AI 幻觉风险** | 无 AI | 双 Agent 互校抗幻觉 |

### 7.2 vs 知识图谱 + AI 题库

| 维度 | 知识图谱 + AI 题库 | ECOS |
|---|---|---|
| **学生模型** | 二元（会/不会）| 多维（5D + Bloom + TC）|
| **教学方法** | 固定知识图谱推题 | Contextual Bandits 最优策略 |
| **误解处理** | 不处理 | 专门有 Misconception 库 |
| **ZPD 建模** | 没有 | 显式建模（跳一跳够得着）|
| **幻觉风险** | 中等 | 双 Agent 互校主动对抗 |

### 7.3 vs LLM 助教（Khanmigo 类）

| 维度 | LLM 助教 | ECOS |
|---|---|---|
| **状态维护** | 每次对话重新来 | 跨会话状态追踪 |
| **幻觉风险** | 高（LLM 直接判断）| 低（CTA 科学估计 + LCA 实验验证）|
| **理论根基** | 无系统理论 | 认知科学（ZPD/CLT/Bloom/CA）|
| **科学实验闭环** | 没有 | 有（CTA→LCA→学生→CTA 互校）|
