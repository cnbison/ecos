## 1. 起点：今天 AI 教育系统的三大局限

ECOS 的应用场景定位，要从"前一代为什么不够"出发。v0.1 综合报告 §第一部分 给出了清晰的"三代演进"框架：

| 代际 | 核心范式 | 代表 | 关键缺陷 |
|---|---|---|---|
| **第一代** | 内容教育（讲授 + 练习 + 考试）| 传统学校、录播课 | 无法个性化 |
| **第二代** | 自适应学习（知识图谱 + 知识追踪）| Squirrel AI、ALEKS | 把学生压缩成"会/不会"——丢失思维过程、策略能力、元认知 |
| **第三代** | AI Tutor（LLM + 问答 + 讲题）| Khanmigo、Duolingo Max、Q-Chat | 每次对话重新认识学生；本质仍是"会回答问题的老师"，不是"理解学生并帮助成长的系统" |

**ECOS 自我定位：第四代——理解学生并帮助成长的系统。** 不是更好的"会答问题的老师"，而是 v0.1 §第二部分 提出的 A→B 闭环系统：

```
A（学生现在是谁）
  ↓ 理解（CTA）
B（学生应该成长成什么样）
  ↓ 干预（LCA）
A'（新状态）
  ↓ ...
```

> 详细论证见 [v0.1 §第一部分](../gpt-dialogues/04-cognitive-digital-twin-v01-report.md) 与 [v2.0 §执行摘要](../deep-research/Cognitive-Digital-Twin-Deep-Research.md)。本文档不重复展开。

## 2. 核心定位（一句话）

> **ECOS = 学生认知数字孪生 + AI 学习教练**
> 基于**双 Agent 共进化架构**（CTA 理解学生 + LCA 改变学生）+ **Bloom 目标空间**（6 层认知层级作为目标坐标系）
> 为 K12 学生提供**基于科学化认知估计 + 自适应干预**的**长期认知陪伴系统**

三个关键词必须同时成立，缺一个就不是 ECOS：

| 关键词 | 含义 | 失去它的后果 |
|---|---|---|
| **科学化认知估计** | CTA 用贝叶斯 / IRT / BKT / DKT 等心理测量学方法估计学生状态 | 退化为 LLM 直觉判断，易幻觉 |
| **自适应干预** | LCA 基于 CTA 状态选择最优干预（不是固定知识图谱）| 退化为"题海 + 错题本" |
| **长期认知陪伴** | 持续 6~12 年维护学生画像、轨迹、学习基因 | 退化为单次会话助手 |

## 3. 目标用户三角

ECOS 不是单边产品，而是 K12 教育场景下**三方用户**的协同系统：

```
                    ┌─────────────┐
                    │   学生（主） │
                    └──────┬──────┘
                           │ 日常使用
                           ↓
        ┌──────────┐   ECOS   ┌──────────┐
        │ 教师（次）│ ←──┐     │ 家长（辅）│
        └──────────┘    │     └──────────┘
                    阶段性协作
```

### 3.1 主用户：K12 学生

| 维度 | 定位 |
|---|---|
| **学段** | 小学高年级（4-6 年级）~ 高中（高三） |
| **使用场景** | 课后作业、复习备考、概念理解、错题反思 |
| **使用频率** | 每日 30~90 分钟（与学习强度相关）|
| **核心诉求** | "我哪里不行？" "下一步学什么？" "我进步了吗？" |
| **与 ECOS 关系** | 数据产生者 + 干预接受者 |

### 3.2 次用户：教师

| 维度 | 定位 |
|---|---|
| **使用场景** | 班级整体学情诊断、学生个体关注重点、阶段性教学调整 |
| **使用频率** | 每周 1~3 次 |
| **核心诉求** | "这周班级该重点讲什么？" "哪些学生需要额外关注？" |
| **与 ECOS 关系** | 阶段性查阅 CTA 信念 + LCA 推荐，反哺课堂教学 |

### 3.3 辅用户：家长

| 维度 | 定位 |
|---|---|
| **使用场景** | 阶段性了解孩子学习状况、参与学习目标设定 |
| **使用频率** | 每月 1~4 次 |
| **核心诉求** | "我家孩子最近学得怎么样？" "该不该报辅导班？" |
| **与 ECOS 关系** | 接收 CTA 信念的可视化 + 阶段性成长报告 |

> **设计原则**：产品形态以学生端为主，教师/家长端为辅；不可本末倒置（不能让家长端成为日常使用入口）。

## 4. 四大核心应用场景

ECOS 的所有具体功能，归结为**四个相互嵌套的应用场景**。每个场景对应一个或多个 ECOS 核心能力。

### 场景 A：学科诊断

**用户痛点**：中国 K12 学生最常见的元认知问题——"我数学哪里不行？"。学生能看到"错题"，但不知道为什么错、卡在哪个认知层级、属于知识缺口还是程序错误。

**ECOS 解决方式**：

```
学生提问："我二次函数应用题总错"
        ↓
CTA 信念分布输出：
  K(二次函数概念)        : {掌握: 0.85, 置信度: 0.9}
  P(应用题解题程序)      : {掌握: 0.35, 置信度: 0.85}
  C(二次函数 ↔ 实际建模) : {掌握: 0.40, 置信度: 0.7}
  S(读题策略)             : {掌握: 0.55, 置信度: 0.6}
  X(几何/代数辅助)        : {掌握: 0.70, 置信度: 0.8}
BloomProfile: Apply L3 (基本应用) | Analyze L1 (建模识别弱)
        ↓
LCA 干预建议："先做 3 道'读题 → 识别模型'的练习，建立二次函数与实际情境的映射"
```

**关键差异化**（vs 现有方案）：

| 对比项 | 错题本/作业帮 | Squirrel AI | Khanmigo | **ECOS** |
|---|---|---|---|---|
| 输出粒度 | 错题列表 | 知识点标签 | 自由对话 | **5D 信念分布 + BloomProfile** |
| 状态表达 | 二元（会/不会）| 二元（掌握/未掌握）| 无状态 | **带置信度的连续分布** |
| 归因深度 | 不归因 | 知识点级别 | 无归因 | **K/P/S/C/X 多维归因** |
| Bloom 层级 | 不涉及 | 不涉及 | 不涉及 | **6 层坐标系显式建模** |

> 工程支撑：[v2.0 §3.3 CTA — State Estimator](../deep-research/Cognitive-Digital-Twin-Deep-Research.md) / [v0.1 §第四部分](../gpt-dialogues/04-cognitive-digital-twin-v01-report.md)

### 场景 B：自适应干预

**用户痛点**：题海战术 vs 精准突破的两难——做多了低效，做少了怕漏。学生无法判断"接下来该练什么"。

**ECOS 解决方式**：

LCA 不是"按知识图谱推荐下一题"，而是基于 CTA 状态选择**最优干预策略**：

```
CTA 状态 → LCA 策略空间 → 选择最优
  ↓          ↓                ↓
"程序技能弱"  [解释型, 练习型, 探究型, 反馈型, 苏格拉底型]
              [参数: 难度, 数量, 反馈密度, 例题风格]
                  ↓
              例如：练习型 + 难度 L3 + 数量 5
                    + 例题按 LearningDNA 偏好风格
                    + 每题反馈立即给
                    ↓
              干预执行 + 数据采集
                  ↓
              CTA 更新信念（程序技能从 0.35 → 0.45）
```

**干预的四个核心机制**（来自 [v2.0 §3.5 互校机制](../deep-research/Cognitive-Digital-Twin-Deep-Research.md)）：
1. **CTA 提出假设** —— "学生程序技能弱"
2. **LCA 设计实验验证** —— "先做 3 道程序型题"
3. **观察结果** —— "3 道错了 2 道，错误集中在分情况讨论"
4. **CTA 更新信念 + LCA 重新规划** —— "程序技能 0.35 → 0.30，分情况讨论是子缺口"

**关键差异化**：

| 对比项 | 作业帮/小猿搜题 | Khan Academy | **ECOS LCA** |
|---|---|---|---|
| 推荐逻辑 | 相似题推送 | Mastery 阈值切换 | **CTA 信念驱动的策略优化** |
| 干预粒度 | 单题 | 知识单元 | **5D 状态 × Bloom 层级 × LearningDNA** |
| 反馈机制 | 答案对错 | 视频讲解 | **基于双 Agent 互校的效果归因** |
| 长期优化 | 无 | 知识点路径 | **跨会话强化学习（隐式）**|

> 工程支撑：[v2.0 §3.4 LCA — Policy Optimizer](../deep-research/Cognitive-Digital-Twin-Deep-Research.md) / [v0.1 §第六部分（双 Agent 闭环）](../gpt-dialogues/04-cognitive-digital-twin-v01-report.md)

### 场景 C：长期成长轨迹

**用户痛点**："3 个月 / 1 学期 / 1 年 / 3 年我进步了多少？"——学生难以感知认知层面的慢变化，家长更无从判断。教育系统目前只能给"分数曲线"，但分数不能反映认知结构演化。

**ECOS 解决方式**：

ECOS 的**数据资产壁垒**——持续 6~12 年维护学生的三维成长画像：

```
┌─────────────────────────────────────────────────────────┐
│           学生 X 的成长画像（示例：初二 → 高一）            │
├─────────────────────────────────────────────────────────┤
│ 1. 5D 状态轨迹（K/P/S/C/X 跨学段演化曲线）              │
│ 2. BloomProfile 演化（记忆→应用→分析 → 跨入"创造"萌芽） │
│ 3. LearningDNA（视觉型/听觉型/动觉型 → 偏好稳态）        │
│ 4. 关键转折点（如：初二数学分析能力突破、高一策略重组）   │
└─────────────────────────────────────────────────────────┘
```

**关键差异化**（护城河）：
- **3 年以上纵向认知数据**目前无人系统性积累——Squirrel AI / Khan Academy 有行为数据但**没有"认知状态轨迹"**
- **跨学科、跨学段的认知演化模型**是 ECOS 独有
- **LearningDNA 跨学段稳定性**研究是无人区（哪些学习基因持久？哪些随认知发展变化？）

**对学生的价值**：可视化"我每一步怎么成长的"——建立成长型思维（growth mindset）所需的"过程感"
**对教师/家长的价值**：阶段性格局判断——"这学期是认知重组期，需要降低对分数的焦虑"

> 工程支撑：[v2.0 §3.3 GrowthTrajectory](../deep-research/Cognitive-Digital-Twin-Deep-Research.md) / [v0.1 §第十部分](../gpt-dialogues/04-cognitive-digital-twin-v01-report.md) / [v0.1 §第五部分 LearningDNA](../gpt-dialogues/04-cognitive-digital-twin-v01-report.md)

### 场景 D：教师/家长协作

**用户痛点**：教师面对 30-50 人的班级，无法精细关注每个学生；家长面对孩子学习只能看到"分数"，看不到认知结构。

**ECOS 解决方式**：

CTA 信念的**可解释性输出**——不是"诊断结果"，而是"诊断依据 + 不确定性 + 建议方向"：

```
┌──────────────────────────────────────────────────┐
│ 给教师的报告（每周）                               │
├──────────────────────────────────────────────────┤
│ 班级整体：                                        │
│   - 二次函数应用题平均程序技能 0.45（年级中位 0.55）│
│   - 主要卡点：分情况讨论（62% 学生弱）              │
│   - 建议：下周增加 1 节"分情况建模"专题课           │
│                                                   │
│ 重点学生（5 名）：                                 │
│   - 张三：程序技能 0.25（极弱），可能需要 1v1      │
│   - 李四：知识 0.90 但程序 0.35（典型"会做不会讲"）│
└──────────────────────────────────────────────────┘
```

**关键差异化**：

| 对比项 | 班级优化大师/极课大数据 | 一起作业/学而思家长端 | **ECOS 教师/家长端** |
|---|---|---|---|
| 数据来源 | 考试成绩 | 作业完成度 | **CTA 5D 信念 + BloomProfile** |
| 输出逻辑 | 分数排名 / 错误率 | 完成率 / 时长 | **认知状态分布 + 干预建议** |
| 因果深度 | 分数→分数 | 行为→行为 | **行为→认知状态→因果归因** |
| 决策支持 | "谁分数低" | "谁没完成" | **"为什么 + 下一步怎么办"** |

**MVP 阶段不实现**（仅学生端）——见 §7。

## 5. 跨场景核心能力清单（ECOS 必有的能力）

四大场景不是 4 个独立产品，而是**同一组核心能力的不同表达**：

| 核心能力 | 场景 A | 场景 B | 场景 C | 场景 D |
|---|---|---|---|---|
| **CTA 信念分布估计**（5D + BloomProfile）| ✅ 主输出 | ✅ 输入 | ✅ 轨迹数据 | ✅ 解释基础 |
| **LCA 策略优化**（基于 CTA 选干预）| — | ✅ 主输出 | — | ✅ 推荐生成 |
| **双 Agent 互校机制**（抗幻觉）| ✅ 信念归因 | ✅ 干预验证 | ✅ 关键转折点归因 | ✅ 报告可信度 |
| **Bloom 目标空间**（6 层坐标系）| ✅ 层级定位 | ✅ 目标选择 | ✅ 演化维度 | ✅ 报告维度 |
| **LearningDNA**（学习基因）| — | ✅ 干预参数 | ✅ 跨学段稳定性 | — |
| **持久化与会话管理** | ✅ 跨会话状态 | ✅ 跨会话策略 | ✅ 跨学段画像 | ✅ 跨阶段报告 |
| **可解释性**（CTA 信念可对外解释）| — | — | — | ✅ 主输出 |

> 任意一个核心能力缺失，相应场景就退化为"前一代 AI 教育系统"。**完整保留 7 项能力是 ECOS 与所有竞品的根本分水岭。**

## 6. 不做清单（明确边界）

ECOS 战略层最重要的决策之一是**明确不做什么**——研究项目精力有限，必须聚焦：

| 不做 | 理由 |
|---|---|
| ❌ **内容生产**（不写教材、不出题目）| 教育内容生态已被成熟公司占据；ECOS 不与新东方/学而思/人教社竞争内容 |
| ❌ **题库生成**（不替代作业帮/小猿）| 题库是搜索引擎范式，不是 ECOS 范式 |
| ❌ **学科覆盖广度**（Phase 4 仅初中数学；Phase 5 拓展到高中数学+初中物理）| K12 学科差异巨大；MVP 必须先在一个学科验证 CTA/LCA 的有效性 |
| ❌ **实时直播课** | 真人教师不可替代；ECOS 是"认知陪伴"不是"教学交付" |
| ❌ **教师备课工具**（v1 不做）| 教师备课与 ECOS 的核心场景弱相关，优先级低 |
| ❌ **家长社交**（不建立家长社区）| 偏离 ECOS 核心，且社交功能是另一类产品 |
| ❌ **学科外的"通识/兴趣教育"**（不涉猎编程启蒙/艺术/体育等）| 12 年级以下认知模型不能照搬，扩科必须重做 CTA/LCA |
| ❌ **成人教育**（考研/职业培训）| 成人认知状态空间与 K12 学生根本不同（动机/经验/目标差异） |
| ❌ **情感陪伴**（不做心理健康/AI 朋友角色）| 与 ECOS "科学化认知估计"方向冲突——会污染 CTA 信念 |

**为什么"不做"很重要**：ECOS 与竞品的根本差异不是"做得更多"，而是"做得不同"。每一个"不做"都是避免方向漂移的护栏。

## 7. MVP 范围（场景对应）

[v2.0 §执行摘要 产品化路径](../deep-research/Cognitive-Digital-Twin-Deep-Research.md) 已经定义：**MVP = 2-4 周 + 初中数学 + 50-100 学生**。

四个核心场景在 MVP 阶段的对应：

| 场景 | MVP 阶段 | 范围 |
|---|---|---|
| **A 学科诊断** | ✅ **必含** | 初中数学（代数 + 几何）+ 5D 状态 + BloomProfile L1-L4 |
| **B 自适应干预** | ✅ **必含** | 基于 CTA 状态选择下一题 / 下一讲解 / 下一练习 |
| **C 长期成长轨迹** | ⚠️ **部分** | 仅学期内（约 4 个月），不跨学期不跨学段 |
| **D 教师/家长协作** | ❌ **不含** | 仅学生端；教师/家长端留到 Phase 5（产品化阶段） |

**MVP 阶段的核心假设验证**（来自 v2.0）：
1. CTA 5D 状态是否能在 50-100 学生上稳定预测（准确度 vs 传统 IRT/BKT）
2. Bloom 目标空间在初中数学上的可行性（6 层是否需要重组？L1-L4 是否够用？）
3. 双 Agent 互校是否能有效减少 LLM 幻觉（实验对比：单 Agent vs 双 Agent 信念质量）

> Phase 5（产品化，2-3 月）→ Phase 6（平台化，6-12 月）的场景扩展详见 [03-roadmap.md](03-roadmap.md)（待写）。

                │
        不改变 ─┼─ 改变
