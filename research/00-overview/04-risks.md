| 等级 | 含义 | 处理原则 |
|---|---|---|
| **🔴 高** | 可能导致 Phase 失败或法律风险 | 必须有显式缓解策略 + 应急预案 |
| **🟡 中** | 可能导致 Phase 延期或体验下降 | 缓解策略必须明确，应急预案可选 |
| **🟢 低** | 不影响 Phase 进度 | 监控 + 被动响应 |

风险等级与 [03-roadmap.md §2-4](03-roadmap.md) 的 H1-H7 假设验证绑定：

```
假设失败 → 对应风险触发

H2 失败 → B1 风险触发（Bloom 6 层不适用）
H3 失败 → A4 风险触发（双 Agent 互校失效）
H5 失败 → E1 + E2 风险触发（商业模式不可行）
H7 失败 → E4 风险触发（数据护城河形成过慢）
```

## A. 技术风险

### A1. 双 Agent 工程复杂度 🔴

**触发条件**：
- CTA 与 LCA 接口不稳定
- 互校循环引入额外延迟（>2 秒）
- 双 Agent 状态同步失败

**影响评估**：
- MVP 延期 2-4 周（占位文件提到）
- 系统稳定性下降（崩溃率 > 5%）
- LLM API 成本翻倍（互校需要多次 LLM 调用）

**缓解策略**：
- **MVP 阶段用伪双 Agent 起步**：CTA 与 LCA 在同一进程中，通过函数调用而非消息队列通信
- **接口标准化**：用 Pydantic 定义 CTA → LCA 的消息契约（避免运行时类型错误）
- **超时保护**：LCA 调用超时 > 30s 自动降级为单 Agent 模式
- **单元测试覆盖**：CTA/LCA 接口单元测试 ≥ 90% 覆盖

**应急预案**：
- 若 MVP 阶段双 Agent 不可行 → 退化为单 Agent + 人工审核层
- 若互校循环延迟过高 → 引入异步互校（不阻塞实时干预）

**监控指标**：
- 双 Agent 接口错误率 ≤ 0.1%
- 互校循环延迟 P95 ≤ 5 秒
- LLM API 调用次数/学生/天 ≤ 50

---

### A2. CTA 5D 状态预测精度不足 🔴（对应 H1）

**触发条件**：
- CTA 5D 状态预测的 AUC < 0.75（[03-roadmap.md §2.3 H1 阈值](03-roadmap.md)）
- 5D 维度间相关性过高（Σ_θ 接近奇异矩阵）
- MIRT 参数冷启动失败（新学科/新学生）

**影响评估**：
- H1 验证失败 → M3 必须回溯到 M2 重新设计 CTA
- 学科诊断失去科学性（"伪置信"未被识别）
- 干预推荐基于错误状态 → 学习效果受损

**缓解策略**：
- **多基线对比**：CTA 5D vs IRT 1D vs BKT 单知识点 vs 人工标注（baseline 比较）
- **增量训练**：从 50 学生开始，逐步扩展到 100 → 500 → 1000（避免一次性大规模实验失败）
- **维度可降级**：若 5D AUC 不达标，可降级为 3D（K/P + Bloom）
- **人工校准集**：每月 10% 学生的 CTA 信念由学科专家审核

**应急预案**：
- 若 AUC < 0.65 → CTA 退化为 LLM-only + 教师审核（牺牲科学性换取可用性）
- 若 5D 失败 → 简化为 3D + Bloom（保留 ECOS 核心差异化）

**监控指标**：
- AUC（5D 状态预测下次表现）
- 维度相关性矩阵秩
- 人工审核分歧率 ≤ 15%

---

### A3. LCA 策略可解释性不足 🟡（对应 H3 部分）

**触发条件**：
- LCA 推荐的干预教师无法理解
- 学生/家长质疑"为什么给我这道题"
- 教师在班级管理时无法与 ECOS 推荐一致

**影响评估**：
- 教师/家长信任度下降（产品风险 B2）
- 学生主观体验差（"刷题感"）
- 商业模式 B2B 推广受阻（H5）

**缓解策略**：
- **rationale 输出**：每个干预附带自然语言理由（"你在二次函数的'分情况讨论'较弱，先做 3 道"）
- **教师后台**：教师可查看每个学生的干预历史 + 状态变化曲线
- **可视化**：5D 状态 + BloomProfile + TC 跨越用雷达图/进度条展示
- **可解释性 vs 性能权衡**：MVP 阶段宁可损失 10% 性能也要可解释

**应急预案**：
- 若教师普遍反馈"看不懂" → 增加教师培训 + 简化展示
- 若 rationale 质量差（LLM 解释不清） → 用模板化解释（预定义文本）

**监控指标**：
- 教师/家长满意度 ≥ 70%（问卷）
- rationale 可读性评分 ≥ 4/5
- 干预接受率 ≥ 60%

---

### A4. 双 Agent 互校无法抗幻觉 🔴（对应 H3）

**触发条件**：
- 双 Agent 信念校准度（ECE）> 0.10（[03-roadmap.md H3 阈值](03-roadmap.md)）
- LLM 解释文本中 misconception 命中率 < 0.5
- 互校循环陷入"CTA 错 → LCA 错 → CTA 跟着错"

**影响评估**：
- H3 验证失败 → M3 回溯
- 系统对外解释可信度崩塌
- 与竞品（Khanmigo/Squirrel AI）的科学化优势不复存在

**缓解策略**：
- **硬底线**：数学层 L0-L2 绝不用 LLM（v0.3.0 + v0.4.0 已确立）
- **LLM Critic 边界**：仅感知层 + 解释层 + Misconception 检测
- **人工审核触发**：当 CTA 信念置信度 < 0.6，自动转人工
- **互校对抗幻觉的 3 机制**（v2.0 §3.5）：CTA 保守 vs LCA 主动 + 数学严格 vs 教学法灵活 + 因果归因强制

**应急预案**：
- 若 ECE > 0.15 → 降低 LLM 介入深度（仅在感知层使用）
- 若互校失效 → 引入教师每周签字（牺牲自动化换可信度）

**监控指标**：
- ECE（双 Agent vs 人工标注）
- Misconception 检测 F1 ≥ 0.7
- 互校循环分歧率（CTA 与 LCA 判断不一致的比例）

## B. 产品风险

### B1. Bloom 6 层在 K12 不适用 🟡（对应 H2）

**触发条件**：
- Bloom 6 层在初中数学的方差解释 < 60%（[03-roadmap.md H2 阈值](03-roadmap.md)）
- Evaluate/Create 层（L5-L6）数据稀疏（学生很少达到）
- 跨学科泛化能力差（数学可用，语文失效）

**影响评估**：
- H2 验证失败 → M3 必须回溯
- BloomProfile 失去预测力
- ECOS 与 Squirrel AI 等的差异化减弱

**缓解策略**：
- **MVP 阶段用 L1-L4**：暂不实现 Evaluate/Create（K12 实际很少达到）
- **学科适配**：数学 L1-L4，物理 L1-L4，语文可能只需 L1-L3
- **教师验证**：与学科教师确认 Bloom 分层是否合理

**应急预案**：
- 若 6 层方差解释 < 40% → 简化为 4 层（L1-L4）
- 若 Evaluate/Create 不可达 → 仅实现 L1-L4（MVP）

**监控指标**：
- BloomProfile 方差解释比例 ≥ 60%
- 各层级学生分布（避免 L5-L6 学生 < 5%）

---

### B2. 早期体验差（学生主观）🟡

**触发条件**：
- 学生首次使用后放弃率 > 30%
- 学生反馈"刷题感" / "比作业帮还烦"
- NPS（净推荐值）< 30

**影响评估**：
- 用户留存率下降（影响 H5）
- 口碑传播失效
- B2C 订阅模式不可持续

**缓解策略**：
- **认知学徒制 6 阶段**：避免纯刷题感，包含 Articulation（自我解释）+ Reflection（反思）
- **游戏化元素**：进度条、等级、勋章（不喧宾夺主）
- **首次体验优化**：第一次做题给出非常友好的反馈 + 引导
- **家长端可见**：学生知道"爸妈能看到我进步"

**应急预案**：
- 若放弃率 > 50% → 立即简化 UI + 增加游戏化
- 若 NPS < 20 → 重新设计引导流程

**监控指标**：
- 7 日留存率 ≥ 60%
- 30 日留存率 ≥ 40%
- NPS ≥ 30

---

### B3. 长期数据稀疏 🟡

**触发条件**：
- 单个学生累积数据 < 50 次作答（影响 BKT 校准）
- 跨学期数据缺失（暑假/寒假断档）
- 不同学科数据不平衡（数学多，物理少）

**影响评估**：
- CTA 状态估计精度下降
- 个性化推荐质量差
- 数据资产护城河形成慢（E4）

**缓解策略**：
- **冷启动**：用 K/P/S/C/X 默认先验 + 短期预测试
- **跨学期衰减模型**：长时间未用 ECOS 时按 BKT 衰减更新 P(L)
- **迁移学习**：从其他学生的同类数据初始化
- **不承诺早期个性化**：明确告诉家长/学生"前 4 周效果有限"

**应急预案**：
- 若数据稀疏 → 降低 BloomProfile 粒度（从 6 层到 4 层）
- 若跨学期断档 → 设计"复学测试"快速重新校准

**监控指标**：
- 平均每位学生作答次数/周 ≥ 10
- 跨学期留存率 ≥ 50%
- 数据稀疏学生比例 ≤ 20%

---

### B4. 学生数据采集质量差 🟡

**触发条件**：
- 学生应付式作答（乱选答案）
- 反思日志为空（学生不写）
- 答题时间不真实（秒选）

**影响评估**：
- CTA 状态估计被污染
- LLM Critic misconception 检测失效
- 数据资产价值降低

**缓解策略**：
- **题型多样化**：不能只看对错，看过程（解答步骤）
- **强制反思节点**：每 N 道题必须做一次反思（不是每次）
- **异常检测**：秒选 + 全部选 C 自动标记为低质量
- **质量权重**：低质量数据在 BKT 更新中降权

**应急预案**：
- 若质量问题严重 → 增加人工审核样本
- 若反思日志普遍为空 → 改为选择题形式（结构化反思）

**监控指标**：
- 反思日志完成率 ≥ 50%
- 异常作答比例 ≤ 10%
- 数据质量评分（人工审核样本）≥ 4/5

## C. 教育专业风险

### C1. 教师协作时间成本超预期 🟡

**触发条件**：
- Q 矩阵构建延后 > 2 周
- TC + Misconceptions 库构建延后 > 2 周
- 教师反馈"负担过重"

**影响评估**：
- MVP M2 延期（4-6 周 → 6-8 周）
- 教师不愿长期合作
- 题目库质量下降

**缓解策略**：
- **分阶段协作**：MVP 先 1 个单元 + 1 个班级，逐步扩展
- **协作工具**：开发 Q 矩阵编辑器（教师可视化标注）
- **报酬机制**：教师按工作量获得合理报酬（MVP 期 200-500 元/单元）
- **CTA 辅助**：CTA 自动建议 Q 矩阵（教师审核修正，而非从零构建）

**应急预案**：
- 若教师协作失败 → 雇佣教研员全职构建 Q 矩阵（成本 ↑）
- 若教师负担过重 → 减少 MVP 题目数（50 题 vs 200 题）

**监控指标**：
- 教师协作满意度 ≥ 70%
- Q 矩阵构建按时率 ≥ 80%
- 教师月度投入时间 ≤ 10 小时

---

### C2. 教学法理论与中国 K12 适配性 🟡

**触发条件**：
- Cognitive Load Theory 4 级呈现不适应中国教材结构
- Bjork 测试效应与中国"题海"文化冲突
- 家长不接受"刻意不教"（合意困难）

**影响评估**：
- LCA 推荐被家长拒绝
- 学生行为不服从（拒绝交错练习）
- 教学效果低于预期

**缓解策略**：
- **文化适配**：与中国教师/教研员共同修订 LCA 策略空间
- **家长教育**：用家长报告说明"为什么这样教"（rationale）
- **渐进引入**：先保留中国家长熟悉的"刷题"元素，逐步加入合意困难
- **本土化研究**：与师范院校合作，研究中国 K12 的教学法适配

**应急预案**：
- 若家长大规模拒绝 → 退化为传统刷题 + 评估（保留 CTA 但简化 LCA）
- 若教学效果差 → 暂停引入合意困难，专注基础功能

**监控指标**：
- 家长接受率 ≥ 70%
- 学生服从率 ≥ 60%
- 教学效果（与传统方法对比）≥ +10%

---

### C3. 学科本体构建难度 🟡

**触发条件**：
- 数学 TC 库难以覆盖所有单元（不只是 8 个）
- 物理 Misconceptions 库与教材版本绑定
- 跨学科能力建模失败（如数学 P 与物理 P 迁移）

**影响评估**：
- M4 学科扩展延期
- 评估粒度不够细
- 与 Squirrel AI 等的本体构建竞争失利

**缓解策略**：
- **MVP 聚焦**：M2-M3 仅构建 1 个学科 1 个学期内容
- **教师共建**：TC + Misconceptions 库与教研组共建
- **迭代扩展**：M4-M5 逐步扩展到多学科
- **自动化辅助**：CTA 自动发现新 misconception（基于学生错误模式聚类）

**应急预案**：
- 若构建过慢 → 减少覆盖范围（MVP 仅 1 个单元）
- 若质量差 → 增加人工审核环节

**监控指标**：
- MVP 单元覆盖率 ≥ 80%
- Misconception 库准确性 ≥ 80%
- 跨学科迁移建模准确性 ≥ 60%

## D. 伦理与法律风险

### D1. 未成年人数据合规 🔴

**触发条件**：
- 违反《个人信息保护法》（中国 2021 实施）
- 违反《儿童个人信息网络保护规定》
- 家长投诉 / 监管处罚

**影响评估**：
- 法律风险（罚款 + 业务暂停）
- 声誉风险（教育行业敏感）
- 商业模式无法启动

**缓解策略**：
- **最小化数据采集**：只采集必要的学习数据（不做行为画像）
- **家长明确同意**：注册流程必须家长双重确认
- **数据本地化**：学生数据存中国境内服务器
- **端侧计算**：敏感数据（姓名、学校）端侧处理
- **差分隐私**：聚合数据发布时加入噪声
- **第三方审计**：每年邀请第三方做数据合规审计

**应急预案**：
- 若被监管约谈 → 立即暂停 + 主动汇报 + 整改
- 若发生数据泄露 → 24 小时内通报 + 启动应急响应

**监控指标**：
- 数据合规审计通过率 = 100%
- 家长撤回率 ≤ 5%
- 数据泄露事件数 = 0

---

### D2. 家长控制与透明度 🟡

**触发条件**：
- 家长无法控制 ECOS 推荐内容
- 学生数据对家长不透明
- 家长担忧 AI 取代真人教师

**影响评估**：
- 家长信任度下降（影响 B2C 模式）
- 媒体负面报道风险
- 政策风险（教育部门限制 AI 使用）

**缓解策略**：
- **家长完全可见**：家长端显示学生全部状态 + 干预历史
- **家长否决权**：家长可关闭特定干预类型
- **人工切换**：家长可一键切换到"人工审核模式"
- **透明算法**：CTA/LCA 算法开源（学术公开）+ 决策可追溯

**应急预案**：
- 若家长大规模担忧 → 加强沟通 + 增加人工审核
- 若媒体负面报道 → 主动公开算法 + 邀请第三方评估

**监控指标**：
- 家长端周活跃率 ≥ 30%
- 家长投诉率 ≤ 2%
- 算法透明度评分 ≥ 4/5

---

### D3. 教育部门监管不确定性 🟡

**触发条件**：
- 教育部门出台 AI 教育应用限制
- 双减政策扩展到 AI 教育
- 学校禁令扩展到课外

**影响评估**：
- B2B 模式受阻（学校不合作）
- B2C 模式受家长购买意愿影响
- 商业模式风险加剧

**缓解策略**：
- **政策跟踪**：专人跟踪教育部 + 地方教育局政策
- **合规先行**：所有功能上线前通过合规审查
- **行业协会**：参与 AI 教育行业标准制定
- **学校合作试点**：先与 1-2 所学校建立试点（控制风险）

**应急预案**：
- 若政策收紧 → 调整商业模式（B2B → B2C 为主）
- 若学校禁令 → 转型为"家长付费 + 学生自用"模式

**监控指标**：
- 政策跟踪覆盖率（教育部 + 31 省）= 100%
- 合规审查通过率 = 100%
- 学校合作试点数 ≥ 2

## E. 商业模式风险

### E1. B2C 付费意愿不足 🟡（对应 H5）

**触发条件**：
- B2C 转化率 < 5%
- 4 个月留存率 < 60%（[03-roadmap.md H5 阈值](03-roadmap.md)）
- 客单价 < 50 元/月（远低于预期 99-299 元）

**影响评估**：
- H5 验证失败 → M5 回溯
- B2C 商业模式不可行
- 资金压力

**缓解策略**：
- **免费试用**：前 4 周免费，降低尝试门槛
- **效果可视化**：让家长明确看到"用了 4 周进步 X 分"
- **社交传播**：家长社群 + 推荐奖励
- **客单价分层**：基础版 99 元/月 + 高级版 299 元/月

**应急预案**：
- 若 B2C 转化 < 2% → 转型为 B2B 为主
- 若留存 < 40% → 重新设计价值主张

**监控指标**：
- B2C 转化率 ≥ 5%
- 4 个月留存率 ≥ 60%
- 客单价 ≥ 99 元/月

---

### E2. B2B 决策周期长 🟡

**触发条件**：
- 学校招标周期 > 12 个月
- 学校决策者不买账（"AI 不靠谱"）
- 教育局统一采购延迟

**影响评估**：
- B2B 模式进展慢
- 现金流不稳定
- 团队士气下降

**缓解策略**：
- **决策者公关**：先与教育局关键决策者建立关系
- **试点先行**：免费试点 1-2 所学校，积累案例
- **效果报告**：试点学校学生的"进步曲线"作为说服材料
- **学校品牌效应**：先做 1 所"明星学校"（如重点中学）

**应急预案**：
- 若 B2B 决策周期 > 18 个月 → 转型为 B2C 为主
- 若学校拒绝合作 → 转为培训机构合作（B2B2C）

**监控指标**：
- 学校试点数 ≥ 2（M5 结束）
- 学校签约数 ≥ 5（M7 结束）
- B2B 决策周期 ≤ 9 个月

---

### E3. 竞品压力 🟡

**触发条件**：
- Khanmigo / Squirrel AI / 字节豆包等竞品快速跟进
- 大厂（字节、阿里、腾讯）进入 K12 AI 教育
- 政策风险导致竞品退出（但 ECOS 也退出）

**影响评估**：
- 差异化优势减弱
- 获客成本上升
- 商业模式承压

**缓解策略**：
- **数据资产护城河**：3 年以上纵向认知数据（H7）
- **科学化定位**：CTA + LCA 双 Agent 是 ECOS 独有（竞品无）
- **学术合作**：与师范院校联合发表，建立学术品牌
- **快速迭代**：6-12 月内保持技术领先

**应急预案**：
- 若大厂进入 → 聚焦"科学化"细分市场（不是大众化）
- 若竞品同质化 → 强化数据资产 + 学术品牌

**监控指标**：
- CTA + LCA 双 Agent 专利申请（如果可能）
- 学术论文发表数
- 与竞品的技术差异评分 ≥ 7/10

---

### E4. 数据资产护城河形成过慢 🔴（对应 H7）

**触发条件**：
- 3 年纵向数据 < 5000 学生（[03-roadmap.md H7 阈值](03-roadmap.md)）
- 学生流失率 > 60%/年
- 跨学段数据断裂（初中毕业后数据丢失）

**影响评估**：
- H7 验证失败 → 数据资产护城河不成立
- ECOS 失去长期竞争优势
- 投资价值降低

**缓解策略**：
- **早期数据收集**：从 MVP 开始就重视数据质量（B4 风险缓解）
- **跨学段衔接**：与高中合作，承接 ECOS 初中毕业生
- **数据资产化**：建立数据资产估值模型，定期对外披露
- **学术研究**：用数据发表论文，建立"ECOS 数据"品牌

**应急预案**：
- 若 3 年数据 < 3000 → 聚焦单学段（初中 3 年）+ 接受数据规模小
- 若流失率 > 70% → 重新设计留存机制

**监控指标**：
- 3 年纵向数据规模 ≥ 5000 学生
- 学生年均流失率 ≤ 30%
- 跨学段衔接率 ≥ 40%

---

## F. 风险监控与应对机制

### F1. 风险监控指标看板

**整体监控**：

| 类别 | 关键指标 | 频率 |
|---|---|---|
| **技术** | 双 Agent 接口错误率 / 互校延迟 / LLM API 成本 | 实时 |
| **产品** | 留存率 / NPS / 数据质量评分 | 每周 |
| **教育** | 教师满意度 / 家长接受率 / 教学效果 | 每月 |
| **法律** | 合规审计 / 数据泄露事件 | 每季度 |
| **商业** | 转化率 / 客单价 / 决策周期 | 每月 |

**红线指标**（一旦触发必须立即响应）：
- 数据泄露事件 ≥ 1
- 合规审计不通过
- 7 日留存率 < 30%
- LLM API 成本超预算 200%
- 监管约谈 / 媒体负面报道

### F2. 风险升级流程

```
Level 1（个人处理）
    ↓ 24 小时内未解决
Level 2（团队处理）
    ↓ 72 小时内未解决
Level 3（创始人决策）
    ↓ 影响 Phase 进度
Level 4（暂停 + 回溯）
```

### F3. 应急预案触发条件

| 触发条件 | 应急预案 |
|---|---|
| H1（H3 后）失败：AUC < 0.65 | CTA 退化为 LLM-only + 教师审核 |
| H2（H3 后）失败：Bloom 方差解释 < 40% | 简化为 L1-L4 |
| H3（H3 后）失败：ECE > 0.15 | 降低 LLM 介入深度 |
| 数据泄露 | 24 小时通报 + 应急响应 |
| 监管约谈 | 暂停 + 主动汇报 + 整改 |
| 商业模式连续 2 季度失败 | 商业模式大调整 |

### F4. 风险文档维护

**维护规则**：
- 每个 Milestone（M2-M7）结束后更新风险矩阵
- 触发红线指标立即更新
- 每季度（Q1-Q4）做一次风险评估
- 重大事件（数据泄露 / 监管处罚）立即记录
