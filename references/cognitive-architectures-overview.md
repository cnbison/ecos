# 认知架构综述与 SelfLab 关联分析

作者：Bisen & Claude

日期：2026-06-12

---

## 认知架构的一般定义

认知架构（Cognitive Architecture）是**对人类（或类人）认知过程的计算化抽象模型**。它定义了一组固定的、通用的机制，规定：

- 信息如何被感知、编码、存储和检索
- 知识如何表示和组织
- 决策和行为如何产生
- 学习和适应如何发生
- 各认知模块之间如何协作

核心区分：认知架构不是解决某个具体问题的算法，而是一个**通用的认知基础设施**——类似于操作系统的内核，应用程序（具体行为）运行在它之上。

## 经典认知架构

### 1. SOAR（State, Operator And Result）

- **创建者**：Allen Newell, John Laird, Paul Rosenbloom（1983，CMU）
- **设计哲学**：将所有认知行为统一为"问题空间搜索"——给定状态（State）、算子（Operator）、结果（Result）
- **核心组件**：
  - 工作记忆（Working Memory）
  - 产生式记忆（Production Memory，规则库）
  - 语义记忆 + 情节记忆（后加）
  - 决策周期（Decision Cycle）：偏好选择算子
  - 学习机制：Chunking（将搜索到的解法压缩为新规则）
- **优点**：形式化程度极高、可解释、有统一的学习理论、长期维护良好
- **缺点**：规则库膨胀、知识获取瓶颈（需手工编码大量规则）、对连续感知-动作场景处理笨拙、缺乏原生的神经网络集成

### 2. ACT-R（Adaptive Control of Thought—Rational）

- **创建者**：John Anderson（1993，CMU）
- **设计哲学**：以"理性分析"为核心——认知系统的目标是在环境中最大化报酬，所有学习和决策都遵循贝叶斯/统计最优原则
- **核心组件**：
  - 陈述性记忆（Declarative Memory，以"组块/chunk"为单位，带激活值和噪声）
  - 程序性记忆（Procedural Memory，产生式规则）
  - 缓冲系统（Buffer）：视觉、听觉、手动、目标、检索等独立缓冲区
  - 激活传播（Spreading Activation）：记忆检索由激活值决定
  - 学习：贝叶斯更新、效用学习、Production Compilation
- **优点**：与人类行为数据拟合极好（反应时、错误率）、有数学理论支撑、模块化清晰
- **缺点**：计算开销大（蒙特卡洛模拟）、扩展到复杂社会认知困难、对感知和运动控制的建模较弱

### 3. Global Workspace Theory（GWT）/ LIDA

- **创建者**：Bernard Baars 提出 GWT（1988），Stan Franklin 团队实现为 LIDA（Learning Intelligent Distribution Agent，2007，Memphis 大学）
- **设计哲学**：模拟意识的"全局工作空间"——大量并行的无意识专业模块竞争进入全局广播，被广播到的信息成为"有意识"的，供所有模块使用
- **核心组件**：
  - 感知联想记忆（Perceptual Associative Memory）
  - 情节记忆（Episodic Memory）
  - 全局工作空间（Global Workspace）：竞争-广播机制
  - 过程学习（Procedural Memory）
  - 注意力代码模块（Attention Codelets）
  - 认知周期（Cognitive Cycle）：约 100ms 一个周期
- **优点**：有意识/无意识的区分机制清晰、有神经科学依据、认知周期设计自然
- **缺点**：实现复杂度高、全局带宽瓶颈（所有信息都经过单一工作空间）、缺乏对长期人格和自我认同的专门建模

### 4. CLARION（Connectionist Learning with Adaptive Rule Induction On-line）

- **创建者**：Ron Sun（1997，RPI）
- **设计哲学**：双过程理论——显性知识（符号层）与隐性知识（亚符号层）并存，且可以相互转化
- **核心组件**：
  - 上层：显性知识（符号规则）
  - 下层：隐性知识（神经网络，联结主义）
  - 动作选择层、动机层、元认知层
  - 学习：规则提取（从隐性到显性）、规则内化（从显性到隐性）
- **优点**：显性/隐性知识的交互机制独特、能解释内隐学习、对技能习得有好的建模
- **缺点**：系统复杂度高、各层之间的知识转化机制在实践中难以调优、缺乏大规模验证

### 5. Subsumption Architecture（包容架构）

- **创建者**：Rodney Brooks（1986，MIT）
- **设计哲学**：反符号主义——智能不需要中央表征和推理，而是从行为层直接涌现。低层行为"包容"高层行为（低层可以抑制高层的输出）
- **核心组件**：
  - 分层的行为模块（每层是感知-动作的有限状态机）
  - 层间抑制和抑制关系
  - 无中央世界模型
- **优点**：实时性极强、鲁棒、对动态环境响应快、启发了整个行为机器人学
- **缺点**：无法处理需要规划和推理的任务、不可扩展（层数增加时调试困难）、没有学习机制、没有内部状态和记忆

### 6. Society of Mind（心智社会）

- **创建者**：Marvin Minsky（1986，MIT）
- **设计哲学**：心智不是单一系统，而是由大量"简单代理（agent）"组成的社会，每个代理本身很笨，但通过协作产生智能
- **核心概念**：
  - Agent（简单计算单元）
  - K-lines（知识线，激活相关代理群）
  - 框架（Frames）和脚本（Scripts）
  - 多种记忆类型（transient、permanent、memorial）
- **优点**：极具启发性、为多代理系统和分布式AI奠基、对"自我"的解构（自我是众多代理中的一个假象）
- **缺点**：不是可执行的计算架构，更像哲学框架、缺乏形式化实现细节

### 7. OpenCog

- **创建者**：Ben Goertzel（2008）
- **设计哲学**：通用人工智能（AGI）框架——集成多种学习和推理机制（概率推理、进化学习、神经网络）在统一的知识表示（AtomSpace，超图）之上
- **核心组件**：
  - AtomSpace：超图知识库
  - PLN（Probabilistic Logic Networks）：概率逻辑推理
  - MOSES：进化程序学习
  - ECAN（Economic Attention Networks）：注意力分配
  - OpenCogBot：机器人集成
- **优点**：高度模块化、集成多种AI范式、开源活跃
- **缺点**：各子系统之间的整合困难、缺乏统一的学习理论、实际应用案例有限

### 8. Sigma

- **创建者**：Paul Rosenbloom（2013，USC，SOAR 的主要设计者之一）
- **设计哲学**：将 SOAR 的符号传统与概率图模型（特别是因子图）统一——用"图形模型"作为认知的通用计算基底
- **核心组件**：
  - 因子图（Factor Graph）作为统一表示
  - 边界化（Marginalization）作为核心推理操作
  - 长期记忆：陈述性、情节、程序性
- **优点**：形式化优雅、符号与概率推理统一、继承 SOAR 的工程成熟度
- **缺点**：复杂度高、社区较小、实际部署案例有限

## 关键维度对比

| 维度 | SOAR | ACT-R | GWT/LIDA | CLARION | 包容架构 |
|------|------|-------|----------|---------|---------|
| **知识表示** | 符号规则 | 符号+激活值 | 混合 | 符号+联结 | 行为规则 |
| **学习机制** | Chunking | 贝叶斯/效用 | 过程学习 | 显隐转化 | 无 |
| **记忆结构** | 工作+语义+情节 | 缓冲+陈述+程序 | 全局工作空间 | 双层 | 无 |
| **意识/自我模型** | 无原生支持 | 无原生支持 | 有（全局广播） | 元认知层 | 无 |
| **可解释性** | 高 | 高 | 中 | 中 | 高 |
| **扩展性** | 中 | 中 | 中 | 低 | 低 |
| **神经科学依据** | 弱 | 中 | 强 | 中 | 弱 |

## 对 SelfLab 的启示

### 1. 没有哪个经典架构直接解决了"自我"问题

GWT/LIDA 最接近——它有意识的机制（全局广播），但"意识"≠"自我"。意识是信息整合的机制，自我是身份、价值观、叙事的持续统一体。ACT-R 有情节记忆和激活传播，但没有人格层。SOAR 后期加了语义记忆和情节记忆，但目标是问题求解，不是存在。

### 2. SGE 架构在经典架构之上加了一层"人格涌现层"

- Event Generator ≈ 感知层（所有架构都有）
- Memory Layer ≈ ACT-R 的陈述性记忆 / SOAR 的情节记忆
- Reflection Layer ≈ CLARION 的显隐转化 / ACT-R 的贝叶斯更新
- Value Layer ≈ ACT-R 的效用学习，但更抽象
- Identity + Narrative Layer → **这一层是经典架构中几乎不存在的**

### 3. 最大的空白在"叙事连续性"

经典认知架构关注的是"如何在当下做决策"，而不是"如何讲述一个关于自己的连贯故事"。SGE 的 Narrative Layer 正是在填补这个空白——这可能是 SelfLab 最独特的贡献点。

### 4. 双系统设计借鉴了 CLARION，但触发机制更好

CLARION 的显隐转化是持续的，SGE 的设计用"认知失调"作为触发器，这更高效，也更符合认知科学中"预测误差驱动学习"的理论（Friston 的自由能原理）。
