## 关键决策与批判性修正

### 1. MVP 时间从 v2.0 的 2-4 周修正为 4-8 周

理由：
- 12 个 MVP 组件（CTA 5 层 + LCA 2 层 + 互校 + 持久化 + UI）
- Q 矩阵构建（与教师协作）
- TC + Misconceptions 库构建
- LLM Critic 集成
- UI 开发 + 教师培训 + 学生招募

### 2. 核心假设从 3 个扩展到 7 个

| 假设 | 阶段 | 内容 | 验证阈值 |
|---|---|---|---|
| **H1** | M3 | CTA 5D 状态预测力 | AUC ≥ 0.75（vs IRT baseline 0.65）|
| **H2** | M3 | Bloom 6 层可行性 | 6 层在初中数学的方差解释 ≥ 60% |
| **H3** | M3 | 双 Agent 互校抗幻觉 | 双 Agent ECE ≤ 0.10 |
| **H4** | M4 | 跨学科迁移可行 | 数学 → 物理/英语复用率 ≥ 70% |
| **H5** | M5 | 商业模式可行 | 4 个月留存 ≥ 60% |
| **H6** | M6 | K12 全学段可行 | 学段切换准确率 ≥ 80% |
| **H7** | M7 | 数据资产护城河 | 3 年纵向数据规模 ≥ 5000 学生 |

### 3. 失败回溯路径（v2.0 没有显式）

```
M3 后 H1-H3 失败 → 简化为 3D + 重组 Bloom + 强化规则 → M2'
M5 后 H4-H5 失败 → 聚焦单一学科 + 单一商业模式 → M4'/M5'
```

避免"all-in 单一假设"的陷阱。

### 4. 总时长 32-44 周（理想）/ 36-52 周（保守）

v2.0 原估计 26-52 周（"6-12 月平台化"）—— 本文档更精细：32-52 周，含回溯缓冲。
