### A. 技术风险（4 类）
- **A1** 双 Agent 工程复杂度 🔴
- **A2** CTA 5D 预测精度不足 🔴（对应 H1）
- **A3** LCA 策略可解释性不足 🟡（对应 H3 部分）
- **A4** 双 Agent 互校抗幻觉失败 🔴（对应 H3）

### B. 产品风险（4 类）
- **B1** Bloom 6 层在 K12 不适用 🟡（对应 H2）
- **B2** 早期体验差 🟡
- **B3** 长期数据稀疏 🟡
- **B4** 学生数据采集质量差 🟡

### C. 教育专业风险（3 类）
- **C1** 教师协作时间成本超预期 🟡
- **C2** 教学法理论与中国 K12 适配性 🟡
- **C3** 学科本体构建难度 🟡

### D. 伦理与法律风险（3 类）
- **D1** 未成年人数据合规 🔴
- **D2** 家长控制与透明度 🟡
- **D3** 教育部门监管不确定性 🟡

### E. 商业模式风险（4 类）
- **E1** B2C 付费意愿不足 🟡（对应 H5）
- **E2** B2B 决策周期长 🟡（对应 H5）
- **E3** 竞品压力 🟡
- **E4** 数据资产护城河形成过慢 🔴（对应 H7）

### F. 风险监控与应对机制

- F1 风险监控指标看板（实时/周/月/季度）
- F2 风险升级流程（4 级：个人 → 团队 → 创始人 → 暂停回溯）
- F3 应急预案触发条件
- F4 风险文档维护规则

## 关键设计决策

### 1. 每类风险 5 要素结构化

```
1. 触发条件（什么情况下风险发生）
2. 影响评估（影响哪些 Phase / 指标 / 假设）
3. 缓解策略（如何预防 / 缓解）
4. 应急预案（如果发生怎么办）
5. 监控指标（如何量化风险状态）
```

### 2. 风险与假设的显式对应

| 风险 | 对应假设 |
|---|---|
| A2 | H1 CTA 5D 状态预测力 |
| B1 | H2 Bloom 6 层可行性 |
| A4 | H3 双 Agent 互校抗幻觉 |
| E1 + E2 | H5 商业模式可行 |
| E4 | H7 数据资产护城河 |

### 3. 红线指标（5 个）

一旦触发必须立即响应：
- 数据泄露事件 ≥ 1
- 合规审计不通过
- 7 日留存率 < 30%
- LLM API 成本超预算 200%
- 监管约谈 / 媒体负面报道

### 4. 升级流程（4 级）

```
Level 1（个人处理）
    ↓ 24 小时内未解决
Level 2（团队处理）
    ↓ 72 小时内未解决
Level 3（创始人决策）
    ↓ 影响 Phase 进度
Level 4（暂停 + 回溯）
```
