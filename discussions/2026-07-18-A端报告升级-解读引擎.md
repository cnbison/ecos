# 2026-07-18 A 端报告升级：规则引擎生成个人学习画像 + 修硬编码版本

## 背景

Bisen 用 lbc001 答完 10 道题，导出 `/api/report/lbc001` 后反馈两个问题：

1. `ecos_version` 字段写死 `"0.42.0"`，但实际代码版本已经是 0.46.5 → 硬编码严重漂移
2. 报告对 C 端学生/家长不友好：raw state dict 字段太多，看不出"我学得怎么样"

按 `discussions/2026-07-17-方向选择-A先C后.md` §任务 4，A 端的核心交付物是
**"学生/家长能看懂的个人学习画像"**——必须用纯规则引擎（不调 LLM），保证离线可用。

## 决策

- **不调 LLM**：走规则引擎（threshold + 模板拼接），保证 C 端场景离线/零成本/可解释
- **bump 到 0.47.0**：0.46.4 / 0.46.5 commit 漏 bump `__init__.py`，这次顺便补上
- **后端先落地**：本次只做 data layer（`/api/report` 返回新字段），前端 UI 折叠面板留到下一轮

## 改动

### 1. `web/api/interpretation.py`（新增，290 行）

6 段规则解读：

| 字段 | 内容 | 阈值 |
|------|------|------|
| `overall` | 总评 1 段话 | — |
| `five_d` | 5 维逐项（档位+theta+置信度+评语）| strong ≥ 0.6, weak < 0.3, conf 高 ≥ 0.5 |
| `bloom` | 主导层 + 下一层 gap + 未探及层 | gap < 0 = 未跨层 |
| `tc` | 5 topic 逐项分类 | progress < 0.3 / 0.3-0.7 / ≥ 0.7 |
| `trajectory` | 首末 5D Δ + 显著维度 | 显著阈值 \|Δ\| ≥ 0.15 |
| `next_steps` | 3-5 条可操作建议 | 优先级：补弱 → 跃层 → 拓概念 → 提样本 |

阈值常量集中在文件顶部，方便后续按 Bisen 反馈调参。

### 2. `web/api/app.py` /api_report

- 删 `"ecos_version": "0.42.0"` 硬编码 → 读 `ecos.__version__`
- `import build_interpretation` + try/except 包裹（解读失败不阻塞主报告）
- 新增 `report["interpretation"]` 字段

### 3. `ecos/__init__.py`

- `0.46.3 → 0.47.0`（补 0.46.4/0.46.5 漏 bump + 新功能）
