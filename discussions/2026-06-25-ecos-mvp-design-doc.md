## 核心内容

### 1. MVP 范围

| 维度 | MVP 范围 |
|---|---|
| 学科 | 初中数学（代数 + 几何）|
| 年级 | 初一 + 初二 |
| 学生规模 | 50-100 |
| 时长 | 4-8 周（修正 v2.0 的 2-4 周）|
| 场景 | A 学科诊断 + B 自适应干预 + C 成长轨迹 |
| 教师/家长端 | 不实现 |

### 2. 4 个核心假设（H1-H4）

| 假设 | 阈值 |
|---|---|
| H1 CTA AUC ≥ 0.75 | 5D + BloomProfile 优于 IRT |
| H2 Bloom 方差 ≥ 60% | 6 层在 K12 可行 |
| H3 双 Agent ECE ≤ 0.10 | 互校抗幻觉 |
| H4 元认知提升 ≥ 0.2 | CTA/LCA 分工有效 |

### 3. W1-W8 任务分解

```
W1-W2: 核心模块（CTA + LCA + Bloom + 互校）
W3-W4: 系统集成（持久化 + LLM + UI + 端到端）
W5-W6: 内部测试（教师协作 + 招募 + Beta）
W7-W8: 实验评估（50-100 学生 + H1-H4 验证 + 报告）
```

### 4. 资源需求

- 团队：3.5 FTE（算法 1 + 后端 1 + 前端 1 + 教研 0.5）
- 总预算：~16-32 万（LLM API + 服务器 + 教师 + 学生奖励）

### 5. 关键决策

- MVP 时间：4-8 周（修正 v2.0 的 2-4 周）
- 仅初中数学（K12 学科差异巨大）
- 教师/家长端 MVP 不实现
- 3 组对照（实验 + 仅 CTA + 传统教学）
- 家长书面同意书强制（未成年人合规）

## 关键决策

| 决策 | 选择 |
|---|---|
| MVP 时间 | 4-8 周（修正）|
| 学科范围 | 仅初中数学 |
| 学生规模 | 50-100 |
| 对照组 | 3 组（90 学生）|
| 教师/家长端 | MVP 不实现 |
| 跨学期 | MVP 不实现 |
