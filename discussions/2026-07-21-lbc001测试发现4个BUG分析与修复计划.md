## 0. 背景

Bisen 作为 ECOS 第一个真实用户 (lbc001), 在 v0.51.4 部署后答了 22 道题 (19 对 / 3 错 / 86.36%), 提出 4 个问题:

1. **总置信度一直在 0.55 左右, 是否有问题?**
2. **有过两次错误, 但没有发现教练干预**
3. **5D 的 C 和 X 数值也基本没什么变化**
4. **LearningDNA 似乎没有启用**

我做了数据收集 + 端到端模拟测试, 发现 4 个观察背后实际是 3 个真 BUG + 1 个设计占位问题。

## 2. 4 个问题逐一分析

### Q1: 总置信度 0.55 不变

**根因**: 不是引擎 bug, 是公式被 C/X 维度低 SE 拉低。

```
overall = mean(1/(1+SE for 5D))
= mean(0.557, 0.583, 0.592, 0.504, 0.504)
= 0.548
```

C/X 的 SE 仍是 0.985 → confidence 接近 0.5, 拉低整体均值。

**预期**: 22 道题后 K 应该更高 conf, 但 K SE 仍 0.80 → K theta 涨了但 SE 没大降。

**本质**: 根因在 Q 矩阵 (见 Q3), C/X 维度没收到足够信息量。

---

### Q2: 两次错误没发现教练干预 ⚠️ 找到 2 个真 BUG

**两个独立 BUG 叠加, misconception 检测在生产里彻底坏掉。**

#### BUG 2.1 — engine 内部检测用错库 (设计 bug)

- 位置: `ecos/cta/belief_engine.py:436` `_llm_critic_misconception`
- 调 `detect_with_hits(...)` **没传 `library_str`**
- detector fallback 到默认 `_MISCONCEPTION_LIBRARY_STR` = K12 通用数学库 M1-M30
- 但 belief.py 用的是 `PYTHON_BASICS_MISCONCEPTION_LIBRARY_STR` = Python 库 M1-M8
- **库 ID 错配**, LLM 永远找不到 Python 相关的 M3 (for 循环 off-by-one)

**实测**: 我手动模拟 PB-Q06 错题 + "range(5) 包括 5" 解释, engine.update 内部检测走 K12 库, 找不到 M3, 不 append。

#### BUG 2.2 — belief.py 末尾独立检测结果不写回 state

- 位置: `web/api/belief.py:532-538` (submit_answer)
- 在 `engine.update()` 之外又调一次 `engine.misc_detector.detect(library_str=Python 库)`
- 这次用对了 Python 库, 能找到 M3 (实测 confidence=0.99)
- 但**只把 detection 返回给前端 response**, **没 append 到 `state.C.misconception_hits`**
- 所以 `state.C.misconception_hits` 始终空 → DB `misconception_history` 始终空

**端到端验证**:
```bash
# belief.py 末尾单独检测
detection = engine.misc_detector.detect(...)
# detection.misc_id = "M3" ✓
# detection.confidence = 0.99 ✓
# 但 state.C.misconception_hits 还是 [] (没 append)
# 所以 DB misconception_history 还是 []
```

**为什么 Bisen 22 道题 0 个命中**:
- 前 14 道 `explanation_text` 是空 (v0.49.2 之前代码), 根本进不了检测
- 后 8 道都答对了, LLM 找不出 misconception
- 即使有新错题, BUG 2.1+2.2 也会让命中数据丢失

#### 历史错题为什么没触发?
- #2 #4 #6 三道错题都在前 14 道, **没存 `explanation_text`**
- `belief.py:532 if explanation_text:` 走 False, **根本没调检测** (数据问题, 不是引擎 bug)

---

### Q3: 5D C 和 X 数值几乎没变化 ⚠️ Q 矩阵设计 bug

26 题 Q 矩阵的 C 和 X 维度 `a_specialized` **全部 0.10** (极小):

| pid | K | P | S | C | X | topic |
|-----|---|---|---|---|---|-------|
| PB-Q01 | 0.90 | 0.20 | 0.40 | **0.10** | **0.10** | variables |
| PB-Q05 | 0.20 | 0.30 | 0.90 | **0.10** | **0.10** | loops |
| PB-Q09 | 0.30 | 0.90 | 0.20 | **0.10** | **0.10** | functions |
| ... 26 题全部 C/X = 0.10 |

MIRT 用 `a_specialized` 算每个维度的信息量:
- C 维度: 22 题 × (0.10² = 0.01) = 信息量 0.22 → SE 接近 1.0
- X 维度: 同上
- 22 道 K 主导题, K SE 0.80, K theta 涨到 1.18
- C/X 主导题**根本不存在**, 所以 C/X theta 永远涨不动

**本质**: Q 矩阵是"5D 评估"标签, 但实际只有 3D 在跑。**设计意图完全没落地**。

## 4. 修复计划

### P0 必修 (本周末)

| 任务 | 位置 | 工作量 |
|------|------|--------|
| BUG 2.1 修 | `ecos/cta/belief_engine.py:436` | 1h |
| BUG 2.2 修 | `web/api/belief.py:529-544` | 0.5h |
| CLAUDE.md 加 CI gate | `CLAUDE.md` | 0.5h |
| 端到端验证 | smoke test | 0.5h |

**P0 修复方向**:
- BeliefEngine `__init__` 接受 `misconception_library_str` 参数
- belief.py 构造 engine 时传 `misconception_library_str=PYTHON_BASICS_MISCONCEPTION_LIBRARY_STR`
- engine._llm_critic_misconception 调 `detect_with_hits(library_str=self.misconception_library_str)`
- belief.py 删掉末尾独立调 detect() (统一走 engine.update)
- misconception_hits 自动 append → save_student_state 持久化 → DB misconception_history 填充

### P1 调研 LearningDNA

**先调研能不能从现有数据推断, 再决定修还是标"待启用"**:

可推断字段 (从 22 道题数据):
- `input_preference`: 暂时无 (没记录学生看图/听音/动手的偏好)
- `feedback_preference`: **可推断** (学生答错后是否需要立刻 feedback, 还是延迟? lbc001 答完立即拿到 AI 评判 = "immediate")
- `fatigue_pattern`: **可推断** (答题时间分布, 早上 vs 晚上 vs 深夜, 错误率)
- `error_pattern`: **可推断** (错误集中在哪些 topic, 错误类型)
- `motivation_pattern`: 暂时无 (没记录主动行为)

**初步方案**:
- 如果有 2-3 个字段能从现有数据推断 → 实现 update 逻辑
- 置信度计算: `confidence = min(1.0, inferred_fields / total_fields)`
- 如果只能推断 1 个字段 → dashboard 标 "LearningDNA 部分启用" 显示已推断的字段
- 如果 0 个字段能推断 → 隐藏 LearningDNA 区域, 标 "待启用"

### C/X 维度 a_specialized 重新设计 (Phase 4+ 长期)

**目标**: 让 C (元认知/条件) 和 X (跨域迁移) 维度有专门题目, 5D 真正评估。

**设计思路** (待具体实施):
- 现有 26 题每个 topic (variables/loops/functions/recursion/scope) 保留
- 新增 4-6 题专门测试 C 维度:
  - "条件判断" (if-else 条件嵌套)
  - "错误调试" (trace code 找 bug)
  - "代码解释" (用自己语言解释代码意图)
- 新增 4-6 题专门测试 X 维度:
  - "跨语言类比" (Python 概念类比 JS 概念)
  - "抽象模式" (从具体例子提取模式)

**Q 矩阵扩充计划**:
- 当前 26 题 → 目标 100+ 题 (DoD: 100-200)
- 分配: 5 topic × 6 Bloom = 30 题 (现状), +20 C 维度题, +20 X 维度题, +5 跨学科类比题 = 75-100 题

**P2 改整体置信度公式**: **取消**。如果 C/X 重新设计后, 公式自然合理 (5 维度 confidence 都 0.5+ → overall 0.5+), 不需要加权。
